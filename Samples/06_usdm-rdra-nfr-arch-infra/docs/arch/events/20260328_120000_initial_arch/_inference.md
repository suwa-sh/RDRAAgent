## RDRA/NFR モデル分析結果

### 分析した RDRA 要素

| モデル | 要素数 | 主な特徴 |
|--------|--------|---------|
| BUC | 6業務・8フロー・UC 40件超 | タイマートリガー（バーチャル会議室利用開始/終了）、バッチ処理（精算額計算）、非同期通知（会議URL通知）、外部連携（決済機関精算支払） |
| アクター | 3種（オーナー/利用者/運営） | 外部アクター2種（社外）、社内アクター1種。ロール分離が明確 |
| 外部システム | 1（決済機関） | 決済系。冪等性確保が必須 |
| 情報 | 18エンティティ | 金銭関連（決済・精算・手数料）、個人情報（オーナー・利用者）、画像データ（会議室） |
| 状態 | 8状態モデル・遷移パス30超 | オーナー・会議室・予約・会議室利用・問合せ・精算・鍵・決済。複雑なドメインロジック |
| 条件 | 10件 | キャンセルポリシー・精算ルール・使用許諾条件等。ビジネスルール層の複雑さが高い |

### 参照した NFR グレード

| カテゴリ | 主要Lv | 主な影響 |

| A. 可用性 | 運用時間 Lv4, 冗長化 Lv4, RPO Lv3, RTO Lv3 | 24h運用、N+1冗長自動切替、障害直前RPO → CaaS(k8s) + Multi-AZ RDB |
| B. 性能・拡張性 | 同時接続 Lv3, レスポンス Lv3, スケールアウト Lv2 | KVS キャッシュ、CDN 画像配信、スケールアウト対応 |
