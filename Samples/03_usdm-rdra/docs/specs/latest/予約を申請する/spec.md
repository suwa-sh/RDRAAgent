# 予約を申請する

## 概要

利用者が会議室の利用日時を指定して予約申請を行う。予約申請により予約情報が「申請」状態で作成され、会議室オーナーの審査待ちとなる。

## 関連 RDRA モデル

| モデル種別 | 要素名 | 関連 |
|-----------|--------|------|
| BUC | 会議室予約フロー | このUCを含むBUC |
| アクター | 利用者 | 操作するアクター |
| 情報 | 予約情報 | 作成する予約情報 |
| 状態 | 予約 | 予約状態を「申請」に遷移 |

## E2E 完了条件（BDD）

### 正常系

```gherkin
Feature: 予約を申請する

  Scenario: 会議室の予約を申請する
    Given 利用者「田中太郎」がログイン済み
    And 公開中の会議室「新宿カンファレンスルームA」が存在する
    When 会議室「新宿カンファレンスルームA」に対して利用日時「2026-04-10 10:00〜12:00」で予約申請する
    Then 予約情報が「申請」状態で作成される
    And 予約申請完了メッセージが表示される

  Scenario: 利用可能時間帯内で予約を申請する
    Given 利用者「田中太郎」がログイン済み
    And 会議室「新宿カンファレンスルームA」の利用可能時間帯が「09:00〜21:00」である
    When 利用日時「2026-04-10 10:00〜12:00」で予約申請する
    Then 予約情報が「申請」状態で作成される
```

### 異常系

```gherkin
  Scenario: 過去の日時で予約を申請しようとする
    Given 利用者「田中太郎」がログイン済み
    When 利用日時「2025-01-01 10:00〜12:00」で予約申請する
    Then 「過去の日時は指定できません」とエラーが表示される
    And 予約情報は作成されない

  Scenario: 利用可能時間帯外で予約を申請しようとする
    Given 利用者「田中太郎」がログイン済み
    And 会議室「新宿カンファレンスルームA」の利用可能時間帯が「09:00〜21:00」である
    When 利用日時「2026-04-10 22:00〜24:00」で予約申請する
    Then 「利用可能時間帯外です」とエラーが表示される
    And 予約情報は作成されない

  Scenario: 既に予約が入っている日時で予約を申請しようとする
    Given 利用者「田中太郎」がログイン済み
    And 会議室「新宿カンファレンスルームA」に「2026-04-10 10:00〜12:00」の確定済み予約が存在する
    When 利用日時「2026-04-10 11:00〜13:00」で予約申請する
    Then 「指定日時は既に予約されています」とエラーが表示される
    And 予約情報は作成されない
```
