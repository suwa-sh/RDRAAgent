# 精算額を計算する

## 概要

サービス運営担当者が月末に会議室別の利用履歴から手数料を差し引いたオーナーへの精算額を計算する。精算ルールに基づき、利用料合計から手数料合計を差し引いた金額を精算額として算出する。

## 関連 RDRA モデル

| モデル種別 | 要素名 | 関連 |
|-----------|--------|------|
| BUC | オーナー精算フロー | このUCを含むBUC |
| アクター | サービス運営担当者 | 操作するアクター |
| 情報 | 精算情報 | 更新する情報（精算額の算出結果） |
| 情報 | 利用履歴 | 参照する情報（利用料金の基礎データ） |
| 情報 | 手数料売上 | 参照する情報（差し引く手数料） |
| 条件 | 精算ルール | 適用される条件（手数料差引・月末締め） |
| 状態 | 精算 | 未精算から精算計算済みへの状態遷移 |

## E2E 完了条件（BDD）

### 正常系

```gherkin
Feature: 精算額を計算する

  Scenario: オーナーへの精算額を計算する
    Given サービス運営担当者「山田花子」がログイン済み
    And オーナー「鈴木一郎」の2026年3月の利用料合計が100,000円である
    And 手数料合計が10,000円（手数料率10%）である
    When 精算額計算画面で2026年3月の精算額を計算する
    Then オーナー「鈴木一郎」の精算額が90,000円と算出される
    And 精算状態が「精算計算済み」に遷移する

  Scenario: 複数オーナーの精算額を一括計算する
    Given サービス運営担当者「山田花子」がログイン済み
    And 2026年3月に利用実績のあるオーナーが3名存在する
    When 精算額計算画面で2026年3月の精算額を一括計算する
    Then 3名のオーナーそれぞれの精算額が算出される
    And 全件の精算状態が「精算計算済み」に遷移する

  Scenario: 精算額の内訳を確認する
    Given サービス運営担当者「山田花子」がログイン済み
    And オーナー「鈴木一郎」の2026年3月の精算が計算済みである
    When 精算額計算画面でオーナー「鈴木一郎」の精算内訳を確認する
    Then 精算ID・オーナー名・精算対象月・利用料合計・手数料合計・精算額・精算状態が表示される
```

### 異常系

```gherkin
  Scenario: 利用実績がないオーナーの精算額を計算しようとした場合
    Given サービス運営担当者「山田花子」がログイン済み
    And オーナー「田中太郎」の2026年3月の利用履歴が0件である
    When 精算額計算画面で2026年3月の精算額を計算する
    Then オーナー「田中太郎」は精算対象外として表示される

  Scenario: 既に精算計算済みの月を再計算しようとした場合
    Given サービス運営担当者「山田花子」がログイン済み
    And オーナー「鈴木一郎」の2026年3月の精算状態が「精算計算済み」である
    When 精算額計算画面で2026年3月の精算額を再計算しようとする
    Then 「既に精算計算済みです。再計算する場合は確認してください」と警告が表示される
```
