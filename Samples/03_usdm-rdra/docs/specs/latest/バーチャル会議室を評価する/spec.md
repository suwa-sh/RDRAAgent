# バーチャル会議室を評価する

## 概要

利用者がバーチャル会議室の接続安定性・音質・操作性などを評価登録する。バーチャル会議室の品質を可視化し、他の利用者の意思決定支援に利用される。

## 関連 RDRA モデル

| モデル種別 | 要素名 | 関連 |
|-----------|--------|------|
| BUC | 会議室評価フロー | このUCを含むBUC |
| アクター | 利用者 | 操作するアクター |
| 情報 | 会議室評価 | 登録する評価の情報 |
| バリエーション | 評価種別 | 会議室評価、オーナー評価、利用者評価 |

## E2E 完了条件（BDD）

### 正常系

```gherkin
Feature: バーチャル会議室を評価する

  Scenario: バーチャル会議室の評価を登録する
    Given 利用者「田中太郎」がバーチャル会議室「オンライン会議室A」を利用済みである
    When 会議室評価登録画面で評価スコア「4」、コメント「接続が安定しており音質も良好でした」を入力する
    And 評価登録ボタンを押下する
    Then バーチャル会議室「オンライン会議室A」に対する会議室評価が登録される
    And 評価の会議室種別が「バーチャル」で保存される

  Scenario: 評価スコア5で最高評価を登録する
    Given 利用者「田中太郎」がバーチャル会議室「Teamsルーム1」を利用済みである
    When 評価スコア「5」、コメント「操作性が抜群で録画機能も便利でした」を入力する
    And 評価登録ボタンを押下する
    Then 評価スコア「5」の会議室評価が登録される
```

### 異常系

```gherkin
  Scenario: 利用していないバーチャル会議室は評価できない
    Given 利用者「田中太郎」がバーチャル会議室「オンライン会議室B」を利用していない
    When バーチャル会議室「オンライン会議室B」の評価を試みる
    Then 「この会議室を利用していないため評価できません」とエラーメッセージが表示される

  Scenario: 評価スコアが範囲外の場合にエラーとなる
    Given 利用者「田中太郎」がバーチャル会議室の評価画面を開いている
    When 評価スコア「0」を入力して評価登録ボタンを押下する
    Then 「評価スコアは1〜5の範囲で入力してください」とエラーメッセージが表示される
```
