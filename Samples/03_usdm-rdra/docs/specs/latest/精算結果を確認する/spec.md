# 精算結果を確認する

## 概要

会議室オーナーが精算額や支払状況を確認する。精算情報とオーナー精算のデータをもとに、月ごとの精算結果と支払状態を把握する。

## 関連 RDRA モデル

| モデル種別 | 要素名 | 関連 |
|-----------|--------|------|
| BUC | オーナー精算フロー | このUCを含むBUC |
| アクター | 会議室オーナー | 操作するアクター |
| 情報 | オーナー精算 | 参照する情報（支払金額・支払状態） |
| 情報 | 精算情報 | 参照する情報（精算額・精算状態） |
| 状態 | 精算 | 精算状態の確認（精算計算済み/支払済み） |

## E2E 完了条件（BDD）

### 正常系

```gherkin
Feature: 精算結果を確認する

  Scenario: 当月の精算結果を確認する
    Given 会議室オーナー「鈴木一郎」がログイン済み
    And 2026年3月の精算額が90,000円で精算状態が「支払済み」である
    When 精算結果確認画面にアクセスする
    Then 2026年3月の精算額90,000円・精算状態「支払済み」が表示される

  Scenario: 過去の精算履歴を確認する
    Given 会議室オーナー「鈴木一郎」がログイン済み
    And 2026年1月から3月までの精算履歴が存在する
    When 精算結果確認画面で精算履歴を表示する
    Then 2026年1月・2月・3月の精算対象月・精算額・精算状態が一覧で表示される

  Scenario: 精算結果の内訳を確認する
    Given 会議室オーナー「鈴木一郎」がログイン済み
    And 2026年3月の精算が支払済みである
    When 精算結果確認画面で2026年3月の内訳を確認する
    Then 利用料合計・手数料合計・精算額・支払金額・支払日・支払状態が表示される
```

### 異常系

```gherkin
  Scenario: 精算履歴がない場合
    Given 会議室オーナー「鈴木一郎」がログイン済み
    And 精算履歴が存在しない
    When 精算結果確認画面にアクセスする
    Then 「精算履歴がありません」とメッセージが表示される

  Scenario: 他のオーナーの精算結果にはアクセスできない
    Given 会議室オーナー「鈴木一郎」がログイン済み
    When 他のオーナー「田中太郎」の精算結果にアクセスしようとする
    Then 「アクセス権限がありません」とエラーが表示される
```
