# 精算結果を確認する - フロントエンド仕様

## 変更概要

精算結果確認画面を新規作成する。会議室オーナーが精算額・支払状況の確認および精算履歴の参照を行えるようにする。

## 画面仕様

### 精算結果確認画面

- **URL**: /owner/settlement/result
- **アクセス権**: 会議室オーナー

#### 表示要素

| 要素 | 種別 | 説明 |
|------|------|------|
| 精算履歴一覧 | テーブル | 精算対象月、利用料合計、手数料合計、精算額、精算状態 |
| 内訳リンク | リンク | 各月の精算内訳詳細への遷移 |
| 精算内訳パネル | パネル | 精算ID、精算対象月、利用料合計、手数料合計、精算額、支払金額、支払日、支払状態 |

#### 操作フロー

1. 会議室オーナーが精算結果確認画面にアクセスする
2. 精算履歴一覧で月ごとの精算額・精算状態を確認する
3. 内訳リンクをクリックして精算内訳の詳細を確認する

## ティア完了条件（BDD）

```gherkin
Feature: 精算結果を確認する - フロントエンド

  Scenario: 精算履歴一覧を表示する
    Given 精算結果確認画面にアクセスする
    Then 精算履歴が月ごとに一覧で表示される
    And 各行に精算対象月・精算額・精算状態が表示される

  Scenario: 精算内訳を確認する
    Given 精算結果確認画面に精算履歴が表示されている

    Then 精算内訳パネルに利用料合計・手数料合計・精算額・支払金額・支払日・支払状態が表示される

  Scenario: 精算履歴がない場合
    Given 精算結果確認画面にアクセスする
    And 精算履歴が存在しない
    Then 「精算履歴がありません」とメッセージが表示される
```
