# 鍵の返却を記録する

## 概要

会議室オーナーが利用者から鍵を返却され、鍵の状態を「貸出中」から「保管中」に遷移させるとともに、会議室利用を「利用終了」状態に遷移させるユースケース。会議室利用ポリシーに基づき、鍵の返却をトリガーとして利用終了を記録する。

## 関連 RDRA モデル

| モデル種別 | 要素名 | 関連 |
|-----------|--------|------|
| BUC | 会議室貸出管理フロー | このUCを含むBUC |
| アクター | 会議室オーナー | 操作するアクター |
| 情報 | 鍵 | 更新する鍵の返却状態 |
| 情報 | 会議室利用 | 更新する会議室利用の状態 |
| 状態 | 鍵 | 貸出中→保管中の状態遷移 |
| 状態 | 会議室利用 | 利用中→利用終了の状態遷移 |
| 条件 | 会議室利用ポリシー | 鍵の返却による利用終了のルール |

## E2E 完了条件（BDD）

### 正常系

```gherkin
Feature: 鍵の返却を記録する

  Scenario: 鍵を返却し会議室利用を終了する
    Given 会議室オーナー「山田花子」がログイン済み
    And 会議室「新宿会議室A」の鍵が「貸出中」状態である
    And 会議室利用が「利用中」状態である
    When 会議室オーナー「山田花子」が鍵返却記録画面で鍵の返却を記録する
    Then 鍵の状態が「保管中」に遷移する
    And 会議室利用が「利用終了」状態に遷移する
    And 返却日時が記録される
```

### 異常系

```gherkin
  Scenario: 保管中の鍵を返却しようとする
    Given 会議室オーナー「山田花子」がログイン済み
    And 会議室「新宿会議室A」の鍵が「保管中」状態である
    When 鍵の返却を記録しようとする
    Then エラーメッセージ「この鍵は貸出中ではありません」が表示される
```
