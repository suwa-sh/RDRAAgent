# 退会を申請する - バックエンド仕様

## 変更概要

会議室オーナーの退会申請を受け付け、オーナー情報の審査状態を「退会」に更新し、退会日を記録するAPIを新規作成する。退会時にはオーナーの公開中の会議室を非公開に変更する。

## API 仕様

### 退会申請API

- **メソッド**: POST
- **パス**: /api/owners/me/withdraw
- **認証**: 会議室オーナー（本人のみ）

#### リクエスト

| パラメータ | 型 | 必須 | 説明 |
|-----------|---|------|------|
| reason | string | No | 退会理由（任意入力） |

#### レスポンス

| フィールド | 型 | 説明 |
|-----------|---|------|
| ownerId | string | オーナーID |
| reviewStatus | string | 更新後の審査状態（「退会」） |
| withdrawDate | string | 退会日（ISO 8601形式） |

## データモデル変更

### オーナー情報

| カラム | 型 | 説明 | 変更種別 |
|--------|---|------|---------|
| 審査状態 | VARCHAR | 「登録済み」→「退会」への状態遷移を管理 | 変更（状態遷移） |
| 退会日 | DATE | 退会申請日を記録 | 変更（値の設定） |

### 会議室情報

| カラム | 型 | 説明 | 変更種別 |
|--------|---|------|---------|
| 公開状態 | VARCHAR | 退会オーナーの会議室を「非公開」に変更 | 変更（値の更新） |

## ビジネスルール

- 審査状態が「登録済み」のオーナーのみ退会申請を許可する
- 退会日はAPI実行時の日付を設定する
- 退会時にオーナーが所有する公開中・公開可能の会議室は全て「非公開」に変更する
- 未精算の利用がある場合は退会を拒否する
- 本人以外からの退会申請は認可エラーとする
