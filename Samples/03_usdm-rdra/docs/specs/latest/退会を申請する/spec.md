# 退会を申請する

## 概要

会議室オーナーがサービスからの退会を申請する。退会申請が完了するとオーナーの状態が「登録済み → 退会」に遷移し、退会日が記録される。

## 関連 RDRA モデル

| モデル種別 | 要素名 | 関連 |
|-----------|--------|------|
| BUC | オーナー登録管理フロー | このUCを含むBUC |
| アクター | 会議室オーナー | 操作するアクター |
| 情報 | オーナー情報 | 参照・更新する情報 |
| 状態 | オーナー | 「登録済み → 退会」に遷移 |

## E2E 完了条件（BDD）

### 正常系

```gherkin
Feature: 退会を申請する

  Scenario: 登録済みオーナーが退会を申請する
    Given 会議室オーナー「鈴木一郎」が退会申請画面を表示している
    And オーナー「鈴木一郎」の審査状態が「登録済み」である
    When 退会理由「サービス利用終了のため」を入力して退会申請ボタンを押す
    Then オーナー「鈴木一郎」の審査状態が「退会」に遷移する
    And 退会日が本日日付で記録される
    And 退会完了メッセージが表示される

  Scenario: 退会申請後にオーナーの会議室が非公開になる
    Given 会議室オーナー「鈴木一郎」が退会申請画面を表示している
    And オーナー「鈴木一郎」の審査状態が「登録済み」である
    And オーナー「鈴木一郎」が会議室「会議室A」を公開中である
    When 退会申請ボタンを押して退会を確定する
    Then オーナー「鈴木一郎」の審査状態が「退会」に遷移する
    And 会議室「会議室A」の公開状態が「非公開」に変更される
```

### 異常系

```gherkin
  Scenario: 登録済み以外のオーナーが退会を申請した場合にエラーとなる
    Given 会議室オーナー「田中二郎」の審査状態が「審査待ち」である
    When 退会申請画面にアクセスしようとする
    Then 「登録済みのオーナーのみ退会申請が可能です」とエラーが表示される

  Scenario: 未精算の利用がある場合に退会が制限される
    Given 会議室オーナー「鈴木一郎」が退会申請画面を表示している
    And オーナー「鈴木一郎」に未精算の利用が存在する
    When 退会申請ボタンを押す
    Then 「未精算の利用があるため退会できません。精算完了後に再度お試しください。」とエラーが表示される
    And オーナー「鈴木一郎」の審査状態は変更されない

  Scenario: 退会確認ダイアログでキャンセルした場合に退会しない
    Given 会議室オーナー「鈴木一郎」が退会申請画面を表示している
    When 退会申請ボタンを押して確認ダイアログが表示される
    And 確認ダイアログで「キャンセル」を押す
    Then オーナー「鈴木一郎」の審査状態は「登録済み」のままである
```
