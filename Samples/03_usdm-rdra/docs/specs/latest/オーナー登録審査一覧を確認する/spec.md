# オーナー登録審査一覧を確認する

## 概要

サービス運営担当者がオーナー登録申請の審査状況を一覧で確認・管理する。オーナー登録審査条件に基づき、審査待ちのオーナー申請を把握し、個別の審査画面への導線を提供する。

## 関連 RDRA モデル

| モデル種別 | 要素名 | 関連 |
|-----------|--------|------|
| BUC | サービス運営管理フロー | このUCを含むBUC |
| アクター | サービス運営担当者 | 操作するアクター |
| 情報 | オーナー情報 | 参照する情報 |
| 条件 | オーナー登録審査条件 | 適用される条件 |

## E2E 完了条件（BDD）

### 正常系

```gherkin
Feature: オーナー登録審査一覧を確認する

  Scenario: 審査待ちのオーナー一覧を確認する
    Given サービス運営担当者「管理者A」がオーナー審査管理画面を表示している
    And オーナー「鈴木一郎」の審査状態が「審査待ち」である
    And オーナー「田中二郎」の審査状態が「審査待ち」である
    And オーナー「佐藤花子」の審査状態が「登録済み」である
    When 審査状態「審査待ち」でフィルタリングする
    Then オーナー「鈴木一郎」と「田中二郎」が一覧に表示される
    And オーナー「佐藤花子」は一覧に表示されない

  Scenario: 全審査状態のオーナー一覧を確認する
    Given サービス運営担当者「管理者A」がオーナー審査管理画面を表示している
    And オーナー「鈴木一郎」の審査状態が「審査待ち」である
    And オーナー「佐藤花子」の審査状態が「登録済み」である
    And オーナー「山田三郎」の審査状態が「却下」である
    When フィルタを「全て」に設定する
    Then オーナー「鈴木一郎」「佐藤花子」「山田三郎」が一覧に表示される

  Scenario: 一覧から個別審査画面に遷移する
    Given サービス運営担当者「管理者A」がオーナー審査管理画面を表示している
    And オーナー「鈴木一郎」の審査状態が「審査待ち」である
    When オーナー「鈴木一郎」の行をクリックする
    Then オーナー「鈴木一郎」のオーナー審査画面に遷移する
```

### 異常系

```gherkin
  Scenario: サービス運営担当者以外のアクターがアクセスした場合にエラーとなる
    Given 会議室オーナー「鈴木一郎」がオーナー審査管理画面にアクセスしようとする
    When 画面を表示しようとする
    Then 「アクセス権限がありません」とエラーが表示される

  Scenario: 審査対象のオーナーが存在しない場合
    Given サービス運営担当者「管理者A」がオーナー審査管理画面を表示している
    And 審査待ちのオーナーが0件である
    When 審査状態「審査待ち」でフィルタリングする
    Then 「該当するオーナー申請はありません」とメッセージが表示される
```
