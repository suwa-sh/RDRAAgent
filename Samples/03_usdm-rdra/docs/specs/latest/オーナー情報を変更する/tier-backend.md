# オーナー情報を変更する - バックエンド仕様

## 変更概要

会議室オーナーが自身のオーナー情報を更新するAPIを新規作成する。登録済み状態のオーナーのみ更新可能とする。

## API 仕様

### オーナー情報取得API

- **メソッド**: GET
- **パス**: /api/owners/me
- **認証**: 会議室オーナー（本人のみ）

#### リクエスト

なし（認証トークンからオーナーIDを特定）

#### レスポンス

| フィールド | 型 | 説明 |
|-----------|---|------|
| ownerId | string | オーナーID |
| name | string | 氏名 |
| contact | string | 連絡先 |
| email | string | メールアドレス |
| registrationDate | string | 登録日 |
| reviewStatus | string | 審査状態 |

### オーナー情報更新API

- **メソッド**: PUT
- **パス**: /api/owners/me
- **認証**: 会議室オーナー（本人のみ）

#### リクエスト

| パラメータ | 型 | 必須 | 説明 |
|-----------|---|------|------|
| name | string | Yes | 氏名 |
| contact | string | Yes | 連絡先 |
| email | string | Yes | メールアドレス |

#### レスポンス

| フィールド | 型 | 説明 |
|-----------|---|------|
| ownerId | string | オーナーID |
| name | string | 更新後の氏名 |
| contact | string | 更新後の連絡先 |
| email | string | 更新後のメールアドレス |

## データモデル変更

### オーナー情報

| カラム | 型 | 説明 | 変更種別 |
|--------|---|------|---------|
| 氏名 | VARCHAR | オーナーの氏名 | 変更（値の更新） |
| 連絡先 | VARCHAR | オーナーの電話番号 | 変更（値の更新） |
| メールアドレス | VARCHAR | オーナーのメールアドレス | 変更（値の更新） |

## ビジネスルール

- 審査状態が「登録済み」のオーナーのみ情報変更を許可する
- オーナーIDと登録日は変更不可とする
- メールアドレスはオーナー情報テーブル内で一意でなければならない（自身を除く）
- 氏名、連絡先、メールアドレスは全て必須項目とする
- 本人以外のオーナー情報は更新不可とする

```gherkin
    Then ステータスコード409が返される
    And エラーメッセージに「このメールアドレスは既に使用されています」が含まれる

  Scenario: 登録済み以外のオーナーからの更新リクエストはエラーとなる
    Given オーナー「田中二郎」の審査状態が「審査待ち」である
    When PUT /api/owners/me に氏名「田中太郎」を送信する
    Then ステータスコード403が返される
    And エラーメッセージに「登録済みのオーナーのみ情報変更が可能です」が含まれる
```
