# 会議室を登録する

## 概要

会議室オーナーが所有する会議室の物件情報（会議室名、所在地、広さ、収容人数、価格、設備・機能、画像）を登録し、貸し出しの準備を行うユースケース。登録直後の会議室は非公開状態となる。

## 関連 RDRA モデル

| モデル種別 | 要素名 | 関連 |
|-----------|--------|------|
| BUC | 会議室管理フロー | このUCを含むBUC |
| アクター | 会議室オーナー | 操作するアクター |
| 情報 | 会議室情報 | 登録する情報 |
| 状態 | 会議室 | 登録により「非公開」状態が生成される |

## E2E 完了条件（BDD）

### 正常系

```gherkin
Feature: 会議室を登録する

  Scenario: 会議室オーナーが新しい会議室を登録する
    Given 会議室オーナー「山田太郎」がログイン済み
    When 会議室登録画面で以下の情報を入力して登録する
      | 項目       | 値                          |
      | 会議室名   | 渋谷ミーティングルームA       |
      | 所在地     | 東京都渋谷区渋谷1-1-1        |
      | 広さ       | 30㎡                        |
      | 収容人数   | 10名                        |
      | 価格       | 3000円/時間                  |
      | 設備・機能 | プロジェクター, Wi-Fi, 空調   |
    Then 会議室「渋谷ミーティングルームA」が登録される
    And 会議室の公開状態が「非公開」になる

  Scenario: 画像を含む会議室情報を登録する
    Given 会議室オーナー「山田太郎」がログイン済み
    When 会議室登録画面で会議室情報を入力し画像を3枚アップロードして登録する
    Then 会議室が画像付きで登録される
    And 会議室の公開状態が「非公開」になる
```

### 異常系

```gherkin
  Scenario: 必須項目が未入力の場合に登録できない
    Given 会議室オーナー「山田太郎」がログイン済み
    When 会議室登録画面で会議室名を空欄のまま登録を試みる
    Then 「会議室名は必須です」とエラーが表示される
    And 会議室は登録されない

  Scenario: 収容人数に不正な値を入力した場合にエラーとなる
    Given 会議室オーナー「山田太郎」がログイン済み
    When 会議室登録画面で収容人数に「-5」を入力して登録を試みる
    Then 「収容人数は1以上の整数を入力してください」とエラーが表示される

  Scenario: 未認証のユーザーは会議室を登録できない
    Given ログインしていない状態
    When 会議室登録画面にアクセスする
    Then ログイン画面にリダイレクトされる
```
