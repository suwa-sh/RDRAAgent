# 利用状況を分析する - フロントエンド仕様

## 変更概要

利用状況分析画面を新規作成する。サービス運営担当者が利用履歴集計区分（会員別、物件別、期間別）を切り替えて利用状況データを集計・可視化できるようにする。

## 画面仕様

### 利用状況分析画面

- **URL**: /admin/usage-analysis
- **アクセス権**: サービス運営担当者

#### 表示要素

| 要素 | 種別 | 説明 |
|------|------|------|
| 利用履歴集計区分セレクタ | セレクトボックス | 会員別、物件別、期間別を選択 |
| 対象期間（開始） | 日付入力 | 分析対象期間の開始日 |
| 対象期間（終了） | 日付入力 | 分析対象期間の終了日 |
| 検索ボタン | ボタン | 条件に基づき利用状況を検索・集計 |
| 利用状況一覧 | テーブル | 集計区分に応じた利用回数・利用時間・利用料金 |
| 売上実績サマリ | テーブル | 会議室ごとの利用回数・売上金額の実績 |
| CSVエクスポートボタン | ボタン | 表示中の分析結果をCSVでダウンロード |

#### 操作フロー

1. サービス運営担当者が利用状況分析画面にアクセスする
2. 利用履歴集計区分（会員別/物件別/期間別）を選択する
3. 対象期間を指定する
4. 検索ボタンを押下する
5. 選択した集計区分に応じた利用状況がテーブルに表示される
6. 売上実績サマリで収益状況を確認する
7. 必要に応じてCSVエクスポートボタンでデータをダウンロードする

## ティア完了条件（BDD）

```gherkin
Feature: 利用状況を分析する - フロントエンド

  Scenario: 集計区分を切り替えて表示する
    Given 利用状況分析画面が表示されている
    When 利用履歴集計区分を「会員別」から「物件別」に変更する
    Then テーブルのヘッダーが物件別集計の表示形式に切り替わる

  Scenario: 対象期間を指定して検索する
    Given 利用状況分析画面が表示されている
    And 利用履歴集計区分「期間別」が選択されている
    When 対象期間を「2026-01-01」から「2026-03-31」に設定し検索ボタンを押下する
    Then 2026年1月から3月の期間別利用状況一覧が表示される

  Scenario: データが0件の場合のメッセージ表示
    Given 利用状況分析画面が表示されている
    When データが存在しない期間を指定して検索する
    Then 「該当期間の利用状況データがありません」とメッセージが表示される
```
