# バーチャル会議室利用を終了する

## 概要

バーチャル会議室の利用終了時刻到来で自動的に利用終了状態に遷移する。バーチャル会議室利用ポリシーに基づき、物理会議室のような鍵の返却は不要で、タイマーによる自動遷移で利用が終了される。

## 関連 RDRA モデル

| モデル種別 | 要素名 | 関連 |
|-----------|--------|------|
| BUC | 会議室貸出管理フロー | このUCを含むBUC |
| 情報 | 会議室利用 | 利用状態を管理する情報 |
| 条件 | バーチャル会議室利用ポリシー | バーチャル会議室の利用終了ルール |
| 状態 | 会議室利用 | 「バーチャル会議室利用を終了する」により利用終了に遷移 |

## E2E 完了条件（BDD）

### 正常系

```gherkin
Feature: バーチャル会議室利用を終了する

  Scenario: 利用終了時刻にバーチャル会議室利用が自動終了される
    Given バーチャル会議室「オンライン会議室A」の利用が「利用中」状態である
    And 予約の利用終了日時が「2026-04-01 12:00」である
    When 現在時刻が「2026-04-01 12:00」に到達する
    Then 会議室利用が「利用終了」状態に遷移する
    And 会議室利用の利用終了日時が「2026-04-01 12:00」で記録される

  Scenario: 鍵の返却なしで利用が終了される
    Given バーチャル会議室「オンライン会議室A」の利用が「利用中」状態である
    When 利用終了時刻に到達する
    Then 鍵の返却処理は実行されない
    And 会議室利用が「利用終了」状態に遷移する
```

### 異常系

```gherkin
  Scenario: 利用中でない場合は利用終了しない
    Given バーチャル会議室「オンライン会議室A」の利用が既に「利用終了」状態である
    When 利用終了時刻に到達する
    Then 状態遷移は実行されない

  Scenario: 物理会議室は自動利用終了の対象外
    Given 物理会議室「新宿カンファレンスルームA」の利用が「利用中」状態である
    When 利用終了時刻に到達する
    Then 自動的な利用終了遷移は実行されない
```
