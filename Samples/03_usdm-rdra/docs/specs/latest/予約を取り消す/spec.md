# 予約を取り消す

## 概要

利用者が予約を取り消す。キャンセルポリシーに基づくキャンセル料が適用され、予約状態が「取消」に遷移する。申請状態、確定状態、変更状態のいずれからも取消可能である。

## 関連 RDRA モデル

| モデル種別 | 要素名 | 関連 |
|-----------|--------|------|
| BUC | 会議室予約フロー | このUCを含むBUC |
| アクター | 利用者 | 操作するアクター |
| 情報 | 予約情報 | 取り消す予約情報 |
| 情報 | キャンセルポリシー | 適用されるキャンセルポリシー |
| 条件 | キャンセルポリシー | キャンセル料の発生条件 |
| 状態 | 予約 | 予約状態を「申請」「確定」「変更」から「取消」に遷移 |

## E2E 完了条件（BDD）

### 正常系

```gherkin
Feature: 予約を取り消す

  Scenario: 確定済みの予約を取り消す（キャンセル料なし）
    Given 利用者「田中太郎」がログイン済み
    And 予約「R001」が「確定」状態で存在する（利用日: 2026-04-20）
    And キャンセルポリシーのキャンセル期限が利用日の3日前である
    And 現在日が「2026-04-15」である
    When 予約「R001」を取り消す
    Then 予約「R001」の状態が「取消」に遷移する
    And キャンセル料は「0円」である
    And 予約取消完了メッセージが表示される

  Scenario: 確定済みの予約を取り消す（キャンセル料あり）
    Given 利用者「田中太郎」がログイン済み
    And 予約「R001」が「確定」状態で存在する（利用日: 2026-04-20、利用料金: 4,000円）
    And キャンセルポリシーのキャンセル期限が利用日の3日前、キャンセル料率が50%である
    And 現在日が「2026-04-18」である
    When 予約「R001」を取り消す
    Then 予約「R001」の状態が「取消」に遷移する
    And キャンセル料は「2,000円」である

  Scenario: 申請状態の予約を取り消す
    Given 利用者「田中太郎」がログイン済み
    And 予約「R002」が「申請」状態で存在する
    When 予約「R002」を取り消す
    Then 予約「R002」の状態が「取消」に遷移する
    And キャンセル料は「0円」である

  Scenario: 変更状態の予約を取り消す
    Given 利用者「田中太郎」がログイン済み
    And 予約「R003」が「変更」状態で存在する
    When 予約「R003」を取り消す
    Then 予約「R003」の状態が「取消」に遷移する
```

### 異常系

```gherkin
  Scenario: 既に取消済みの予約を取り消そうとする
    Given 利用者「田中太郎」がログイン済み
    And 予約「R004」が「取消」状態で存在する
    When 予約「R004」を取り消そうとする
    Then 「この予約は既に取り消されています」とエラーが表示される
```
