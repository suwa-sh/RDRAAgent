# 会議URLを通知する

## 概要

バーチャル会議室の予約確定時に、オーナーが設定した会議URLを利用者にメール等で自動通知する。バーチャル会議室利用ポリシーに基づき、予約確定をトリガーとして自動実行される。

## 関連 RDRA モデル

| モデル種別 | 要素名 | 関連 |
|-----------|--------|------|
| BUC | 会議室貸出管理フロー | このUCを含むBUC |
| アクター | 利用者 | 通知を受け取るアクター |
| 情報 | 会議URL | 通知する会議URLの情報 |
| 情報 | 予約情報 | 予約確定をトリガーとする |
| 条件 | バーチャル会議室利用ポリシー | 会議URL自動通知ルール |

## E2E 完了条件（BDD）

### 正常系

```gherkin
Feature: 会議URLを通知する

  Scenario: バーチャル会議室の予約確定時に会議URLが自動通知される
    Given バーチャル会議室「オンライン会議室A」に会議URL「https://zoom.us/j/1234567890」が登録されている
    And 利用者「田中太郎」の予約が「申請」状態である
    When 会議室オーナー「鈴木花子」が予約を許諾する
    Then 予約が「確定」状態に遷移する
    And 利用者「田中太郎」のメールアドレスに会議URL「https://zoom.us/j/1234567890」が通知される

  Scenario: 通知画面で会議URL情報を確認できる
    Given 利用者「田中太郎」のバーチャル会議室予約が確定している
    When 利用者「田中太郎」が会議URL通知画面を開く
    Then 会議室名「オンライン会議室A」が表示される
    And 会議URL「https://zoom.us/j/1234567890」が表示される
    And 予約日時「2026-04-01 10:00〜12:00」が表示される
```

### 異常系

```gherkin
  Scenario: 物理会議室の予約確定時には会議URLは通知されない
    Given 物理会議室「新宿カンファレンスルームA」の予約が「申請」状態である
    When 会議室オーナーが予約を許諾する
    Then 予約が「確定」状態に遷移する
    And 会議URL通知は実行されない

  Scenario: 会議URLの有効期限が切れている場合にエラーとなる
    Given バーチャル会議室「オンライン会議室A」の会議URLの有効期限が「2026-03-01」（期限切れ）である
    And 利用者「田中太郎」の予約が「申請」状態である
    When 会議室オーナー「鈴木花子」が予約を許諾する
    Then 「会議URLの有効期限が切れています。オーナーに更新を依頼してください」とエラーが記録される
```
