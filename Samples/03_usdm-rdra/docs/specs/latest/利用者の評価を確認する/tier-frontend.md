# 利用者の評価を確認する - フロントエンド仕様

## 変更概要

会議室オーナーが予約申請した利用者の過去の利用評価を確認する利用者評価確認画面を新規作成する。

## 画面仕様

### 利用者評価確認画面

- **URL**: /owner/users/{user_id}/ratings
- **アクセス権**: 会議室オーナー

#### 表示要素

| 要素 | 種別 | 説明 |
|------|------|------|
| 利用者基本情報 | テキスト | 利用者の氏名 |
| 平均評価スコア | テキスト | 利用者の全評価の平均スコア |
| 評価一覧 | テーブル | 評価ID、評価したオーナー名、評価スコア、コメント、評価日時 |
| 評価なしメッセージ | テキスト | 評価が0件の場合に表示されるメッセージ |
| 戻るボタン | ボタン | 予約審査画面に戻る |

#### 操作フロー

1. 予約審査画面から利用者の評価確認リンクを押下する
2. 利用者評価確認画面が表示される
3. 利用者の基本情報と平均評価スコアが表示される
4. 過去の評価一覧が時系列で表示される
5. 確認後、戻るボタンで予約審査画面に戻る
