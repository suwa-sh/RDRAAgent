# 精算を実行する - フロントエンド仕様

## 変更概要

精算実行画面を新規作成する。サービス運営担当者が精算計算済みの精算に対して支払実行を行えるようにする。

## 画面仕様

### 精算実行画面

- **URL**: /admin/settlement/execute
- **アクセス権**: サービス運営担当者

#### 表示要素

| 要素 | 種別 | 説明 |
|------|------|------|
| 精算対象月 | 月選択 | 精算対象の年月を選択（YYYY-MM） |
| 精算対象一覧 | テーブル | オーナー名、精算額、精算状態、チェックボックス |
| 全選択チェックボックス | チェックボックス | 精算計算済みの全件を選択 |
| 精算実行ボタン | ボタン | 選択した精算の支払処理を実行 |
| 実行確認ダイアログ | ダイアログ | 精算実行前の確認（対象件数・合計金額表示） |
| 実行結果 | テーブル | オーナー名、精算額、支払状態、決済機関連携ID |

#### 操作フロー

1. サービス運営担当者が精算実行画面にアクセスする
2. 精算対象月を選択する
3. 精算計算済みの精算一覧が表示される
4. 精算実行対象をチェックボックスで選択する
5. 精算実行ボタンを押下する
6. 実行確認ダイアログで対象件数・合計金額を確認し実行する
7. 決済機関を通じた支払処理が実行され結果が表示される

## ティア完了条件（BDD）

```gherkin
Feature: 精算を実行する - フロントエンド

  Scenario: 精算対象一覧を表示する
    Given 精算実行画面が表示されている
    When 精算対象月を「2026-03」に設定する
    Then 精算計算済みのオーナー一覧が表示される

  Scenario: 精算を実行する
    Given 精算実行画面に精算計算済みのオーナーが3名表示されている
    When 全選択チェックボックスをオンにし精算実行ボタンを押下する
    Then 実行確認ダイアログに「3件、合計270,000円の精算を実行しますか？」と表示される
    And 確認ボタンを押下すると支払処理が実行される
    And 実行結果に支払状態が表示される

  Scenario: 決済機関連携失敗時のエラー表示
    Given 精算実行画面で精算を実行する
    When 決済機関との通信に失敗する
    Then 「決済機関との連携に失敗しました。再度実行してください」とエラーが表示される
```
