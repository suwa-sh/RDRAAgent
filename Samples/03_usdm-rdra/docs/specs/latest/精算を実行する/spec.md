# 精算を実行する

## 概要

サービス運営担当者が決済機関を通じてオーナーへ精算額を支払う。精算計算済みの精算情報に基づき、外部の決済機関と連携して実際の支払処理を実行する。

## 関連 RDRA モデル

| モデル種別 | 要素名 | 関連 |
|-----------|--------|------|
| BUC | オーナー精算フロー | このUCを含むBUC |
| アクター | サービス運営担当者 | 操作するアクター |
| 情報 | オーナー精算 | 更新する情報（精算実行結果の記録） |
| 情報 | 精算情報 | 参照する情報（精算額の確認） |
| 条件 | 支払精算ポリシー | 適用される条件（決済方法ごとの処理手順） |
| 状態 | 精算 | 精算計算済みから支払済みへの状態遷移 |
| 状態 | 決済 | 決済手段登録済みから引き落とし済みへの状態遷移 |
| 外部システム | 決済機関 | 連携する外部システム（精算支払連携） |

## E2E 完了条件（BDD）

### 正常系

```gherkin
Feature: 精算を実行する

  Scenario: オーナーへの精算を実行する
    Given サービス運営担当者「山田花子」がログイン済み
    And オーナー「鈴木一郎」の2026年3月の精算状態が「精算計算済み」で精算額が90,000円である
    When 精算実行画面でオーナー「鈴木一郎」の精算を実行する
    Then 決済機関を通じて90,000円の支払処理が実行される
    And 精算状態が「支払済み」に遷移する
    And オーナー精算の支払状態が記録される

  Scenario: 複数オーナーの精算を一括実行する
    Given サービス運営担当者「山田花子」がログイン済み
    And 精算計算済みのオーナーが3名存在する
    When 精算実行画面で一括精算を実行する
    Then 3名のオーナーへの支払処理が決済機関を通じて実行される
    And 全件の精算状態が「支払済み」に遷移する
```

### 異常系

```gherkin
  Scenario: 精算計算済みでない精算を実行しようとした場合
    Given サービス運営担当者「山田花子」がログイン済み
    And オーナー「鈴木一郎」の精算状態が「未精算」である
    When 精算実行画面で精算を実行しようとする
    Then 「精算計算済みの精算のみ実行可能です」とエラーが表示される

  Scenario: 決済機関との連携に失敗した場合
    Given サービス運営担当者「山田花子」がログイン済み
    And オーナー「鈴木一郎」の精算状態が「精算計算済み」である
    When 精算を実行するが決済機関との通信に失敗する
    Then 「決済機関との連携に失敗しました。再度実行してください」とエラーが表示される
    And 精算状態は「精算計算済み」のまま変更されない
```
