# 決済方法を設定する

## 概要

利用者が予約時にクレジットカードまたは電子マネーの決済手段を登録する。決済情報が「決済手段登録済み」状態に遷移し、利用後の引き落とし処理の準備が完了する。

## 関連 RDRA モデル

| モデル種別 | 要素名 | 関連 |
|-----------|--------|------|
| BUC | 会議室予約フロー | このUCを含むBUC |
| アクター | 利用者 | 操作するアクター |
| 情報 | 決済情報 | 登録する決済情報 |
| 条件 | 支払精算ポリシー | 適用される決済ルール |
| 状態 | 決済 | 決済状態を「未登録」から「決済手段登録済み」に遷移 |
| バリエーション | 決済方法 | クレジットカード、電子マネー |

## E2E 完了条件（BDD）

### 正常系

```gherkin
Feature: 決済方法を設定する

  Scenario: クレジットカードで決済方法を設定する
    Given 利用者「田中太郎」がログイン済み
    And 予約「R001」が申請済みで決済情報が「未登録」状態である
    When 決済方法「クレジットカード」を選択しカード番号「4111-1111-1111-1111」、有効期限「2028-12」を入力して登録する
    Then 決済情報が「決済手段登録済み」状態に遷移する
    And 決済方法設定完了メッセージが表示される

  Scenario: 電子マネーで決済方法を設定する
    Given 利用者「田中太郎」がログイン済み
    And 予約「R001」が申請済みで決済情報が「未登録」状態である
    When 決済方法「電子マネー」を選択し電子マネーアカウント情報を入力して登録する
    Then 決済情報が「決済手段登録済み」状態に遷移する
    And 決済方法設定完了メッセージが表示される
```

### 異常系

```gherkin
  Scenario: 無効なクレジットカード番号を入力する
    Given 利用者「田中太郎」がログイン済み
    When 決済方法「クレジットカード」を選択しカード番号「1234-5678-9012-3456」を入力して登録する
    Then 「無効なカード番号です」とエラーが表示される
    And 決済情報は「未登録」状態のままである

  Scenario: 有効期限切れのクレジットカードを登録しようとする
    Given 利用者「田中太郎」がログイン済み
    When 決済方法「クレジットカード」を選択し有効期限「2024-01」を入力して登録する
    Then 「有効期限が切れています」とエラーが表示される
    And 決済情報は「未登録」状態のままである
```
