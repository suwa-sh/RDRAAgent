# オーナー登録を審査する - フロントエンド仕様

## 変更概要

サービス運営担当者がオーナー登録申請の詳細を確認し、承認または却下の判断を行うオーナー審査画面を新規作成する。

## 画面仕様

### オーナー審査画面

- **URL**: /admin/owners/:ownerId/review
- **アクセス権**: サービス運営担当者

#### 表示要素

| 要素 | 種別 | 説明 |
|------|------|------|
| オーナーID | テキスト | 審査対象のオーナーID |
| 氏名 | テキスト | 申請者の氏名 |
| 連絡先 | テキスト | 申請者の連絡先 |
| メールアドレス | テキスト | 申請者のメールアドレス |
| 登録日 | テキスト | 申請日 |
| 審査状態 | テキスト | 現在の審査状態 |
| 承認ボタン | ボタン | オーナー登録を承認する |
| 却下ボタン | ボタン | オーナー登録を却下する |
| 確認ダイアログ | ダイアログ | 承認・却下操作前の確認表示 |

#### 操作フロー

1. サービス運営担当者がオーナー審査画面にアクセスする
2. 申請者のオーナー情報（氏名、連絡先、メールアドレス等）を確認する
3. オーナー登録審査条件に基づき審査を行う
4. 「承認」または「却下」ボタンを押す
5. 確認ダイアログが表示される
6. 確認後、審査結果がAPIに送信される
7. 審査結果が画面に反映される
