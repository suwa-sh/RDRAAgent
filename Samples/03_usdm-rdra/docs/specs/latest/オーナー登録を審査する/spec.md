# オーナー登録を審査する

## 概要

サービス運営担当者がオーナー登録申請を確認し、オーナー登録審査条件に基づいて承認または却下を判断する。承認時はオーナーの状態が「審査待ち → 登録済み」に、却下時は「審査待ち → 却下」に遷移する。

## 関連 RDRA モデル

| モデル種別 | 要素名 | 関連 |
|-----------|--------|------|
| BUC | オーナー登録管理フロー | このUCを含むBUC |
| アクター | サービス運営担当者 | 操作するアクター |
| 情報 | オーナー情報 | 参照・更新する情報 |
| 状態 | オーナー | 「審査待ち → 登録済み」または「審査待ち → 却下」に遷移 |
| 条件 | オーナー登録審査条件 | 審査判定に適用される条件 |

## E2E 完了条件（BDD）

### 正常系

```gherkin
Feature: オーナー登録を審査する

  Scenario: オーナー登録申請を承認する
    Given サービス運営担当者「管理者A」がオーナー審査画面を表示している
    And オーナー「鈴木一郎」の審査状態が「審査待ち」である
    When オーナー「鈴木一郎」の申請内容を確認し「承認」ボタンを押す
    Then オーナー「鈴木一郎」の審査状態が「登録済み」に遷移する

  Scenario: オーナー登録申請を却下する
    Given サービス運営担当者「管理者A」がオーナー審査画面を表示している
    And オーナー「田中二郎」の審査状態が「審査待ち」である
    When オーナー「田中二郎」の申請内容を確認し「却下」ボタンを押す
    Then オーナー「田中二郎」の審査状態が「却下」に遷移する
```

### 異常系

```gherkin
  Scenario: 審査待ち以外のオーナーを審査しようとした場合にエラーとなる
    Given サービス運営担当者「管理者A」がオーナー審査画面を表示している
    And オーナー「佐藤花子」の審査状態が「登録済み」である
    When オーナー「佐藤花子」に対して「承認」操作を実行する
    Then 「審査待ち状態のオーナーのみ審査可能です」とエラーが表示される
    And オーナー「佐藤花子」の審査状態は変更されない

  Scenario: サービス運営担当者以外のアクターが審査を試みた場合にエラーとなる
    Given 会議室オーナー「鈴木一郎」がオーナー審査画面にアクセスしようとする
    When 画面を表示しようとする
    Then 「アクセス権限がありません」とエラーが表示される
```
