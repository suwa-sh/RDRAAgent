# バーチャル会議室利用を開始する - バックエンド仕様

## 変更概要

バーチャル会議室の利用開始時刻到来時に自動的に利用開始状態へ遷移するバッチ処理を新規作成する。

## API 仕様

### バーチャル会議室利用開始バッチ

- **種別**: スケジュールバッチ（毎分実行）
- **トリガー**: 利用開始時刻到来（タイマー）

#### 処理対象

| 条件 | 説明 |
|------|------|
| 会議室種別 | バーチャル |
| 予約状態 | 確定 |
| 利用開始日時 | 現在日時以前 |
| 会議室利用状態 | 未作成 |

#### 処理内容

1. 条件に一致する予約を検索する
2. 会議室利用レコードを作成し、利用状態を「利用開始」に設定する
3. 利用開始日時を記録する

## データモデル変更

### 会議室利用

| カラム | 型 | 説明 | 変更種別 |
|--------|---|------|---------|
| room_type | VARCHAR | 会議室種別（物理/バーチャル）判定用 | 追加 |

## ビジネスルール

- バーチャル会議室利用ポリシーに基づき、鍵の貸出・返却は不要
- 会議室種別が「バーチャル」かつ予約状態が「確定」の予約のみが対象
- 利用開始時刻到来で自動的に利用開始状態に遷移する
- 物理会議室は本バッチの対象外（従来の鍵貸出フローを適用）
