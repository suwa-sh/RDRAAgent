# バーチャル会議室利用を開始する

## 概要

バーチャル会議室の予約確定後、利用開始時刻に自動的に利用開始状態に遷移する。バーチャル会議室利用ポリシーに基づき、物理会議室のような鍵の貸出は不要で、タイマーによる自動遷移で利用が開始される。

## 関連 RDRA モデル

| モデル種別 | 要素名 | 関連 |
|-----------|--------|------|
| BUC | 会議室貸出管理フロー | このUCを含むBUC |
| 情報 | 会議室利用 | 利用状態を管理する情報 |
| 条件 | バーチャル会議室利用ポリシー | バーチャル会議室の利用開始ルール |
| 状態 | 会議室利用 | 「バーチャル会議室利用を開始する」により利用開始に遷移 |

## E2E 完了条件（BDD）

### 正常系

```gherkin
Feature: バーチャル会議室利用を開始する

  Scenario: 利用開始時刻にバーチャル会議室利用が自動開始される
    Given バーチャル会議室「オンライン会議室A」の予約が「確定」状態である
    And 予約の利用開始日時が「2026-04-01 10:00」である
    When 現在時刻が「2026-04-01 10:00」に到達する
    Then 会議室利用が「利用開始」状態に遷移する
    And 会議室利用の利用開始日時が「2026-04-01 10:00」で記録される

  Scenario: 鍵の貸出なしで利用が開始される
    Given バーチャル会議室「オンライン会議室A」の予約が「確定」状態である
    When 利用開始時刻に到達する
    Then 鍵の貸出処理は実行されない
    And 会議室利用が「利用開始」状態に遷移する
```

### 異常系

```gherkin
  Scenario: 予約が確定状態でない場合は利用開始しない
    Given バーチャル会議室「オンライン会議室A」の予約が「取消」状態である
    When 利用開始時刻に到達する
    Then 会議室利用は開始されない

  Scenario: 物理会議室は自動利用開始の対象外
    Given 物理会議室「新宿カンファレンスルームA」の予約が「確定」状態である
    When 利用開始時刻に到達する
    Then 自動的な利用開始遷移は実行されない
```
