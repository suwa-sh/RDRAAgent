# 利用回数を確認する

## 概要

会議室オーナーが自身の会議室の利用回数実績を確認する。売上実績と利用履歴をもとに、会議室ごとの利用回数を把握し運営改善に活用する。

## 関連 RDRA モデル

| モデル種別 | 要素名 | 関連 |
|-----------|--------|------|
| BUC | 利用実績管理フロー | このUCを含むBUC |
| アクター | 会議室オーナー | 操作するアクター |
| 情報 | 売上実績 | 参照する情報（会議室ごとの利用回数・売上金額） |
| 情報 | 利用履歴 | 参照する情報（会議室の利用履歴詳細） |

## E2E 完了条件（BDD）

### 正常系

```gherkin
Feature: 利用回数を確認する

  Scenario: 自身の会議室の利用回数を確認する
    Given 会議室オーナー「鈴木一郎」がログイン済み
    And 会議室「新宿カンファレンスルームA」の2026年3月の利用回数が10回である
    When 利用回数確認画面にアクセスする
    Then 会議室「新宿カンファレンスルームA」の利用回数10回が表示される

  Scenario: 複数の会議室の利用回数を確認する
    Given 会議室オーナー「鈴木一郎」がログイン済み
    And 会議室「新宿カンファレンスルームA」の利用回数が10回である
    And 会議室「渋谷ミーティングルームB」の利用回数が5回である
    When 利用回数確認画面にアクセスする
    Then 会議室「新宿カンファレンスルームA」の利用回数10回が表示される
    And 会議室「渋谷ミーティングルームB」の利用回数5回が表示される

  Scenario: 期間を指定して利用回数を確認する
    Given 会議室オーナー「鈴木一郎」がログイン済み
    And 会議室「新宿カンファレンスルームA」の2026年1月の利用回数が8回である
    When 対象期間を「2026年1月」に指定して利用回数確認画面を表示する
    Then 2026年1月の利用回数8回が表示される
```

### 異常系

```gherkin
  Scenario: 利用実績がない場合
    Given 会議室オーナー「鈴木一郎」がログイン済み
    And 会議室の利用履歴が存在しない
    When 利用回数確認画面にアクセスする
    Then 「利用実績がありません」とメッセージが表示される

  Scenario: 他のオーナーの会議室の利用回数にはアクセスできない
    Given 会議室オーナー「鈴木一郎」がログイン済み
    When 他のオーナー「田中太郎」の会議室の利用回数にアクセスしようとする
    Then 「アクセス権限がありません」とエラーが表示される
```
