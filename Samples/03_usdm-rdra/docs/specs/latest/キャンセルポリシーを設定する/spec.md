# キャンセルポリシーを設定する

## 概要

会議室オーナーがキャンセル料の発生条件や返金ルールを設定するユースケース。キャンセルポリシーは会議室公開条件の一部であり、予約取消時の精算処理に適用される。

## 関連 RDRA モデル

| モデル種別 | 要素名 | 関連 |
|-----------|--------|------|
| BUC | 会議室管理フロー | このUCを含むBUC |
| アクター | 会議室オーナー | 操作するアクター |
| 情報 | キャンセルポリシー | 設定する情報 |

## E2E 完了条件（BDD）

### 正常系

```gherkin
Feature: キャンセルポリシーを設定する

  Scenario: 会議室オーナーがキャンセルポリシーを新規設定する
    Given 会議室オーナー「山田太郎」がログイン済み
    And 会議室「渋谷ミーティングルームA」が登録済み
    And キャンセルポリシーが未設定
    When キャンセルポリシー設定画面で以下の情報を入力して保存する
      | キャンセル期限      | キャンセル料率 |
      | 7日前まで           | 0%             |
      | 3日前まで           | 30%            |
      | 前日まで            | 50%            |
      | 当日                | 100%           |
    And 返金ルールを「キャンセル料を差し引いた金額を決済方法に返金」に設定する
    Then キャンセルポリシーが設定される

  Scenario: 会議室オーナーがキャンセルポリシーを変更する
    Given 会議室オーナー「山田太郎」がログイン済み
    And 会議室「渋谷ミーティングルームA」にキャンセルポリシーが設定済み
    When キャンセルポリシー設定画面で3日前のキャンセル料率を「50%」に変更して保存する
    Then 3日前のキャンセル料率が「50%」に更新される
```

### 異常系

```gherkin
  Scenario: キャンセル料率に不正な値を入力した場合
    Given 会議室オーナー「山田太郎」がログイン済み
    When キャンセルポリシー設定画面でキャンセル料率に「150%」を入力して保存を試みる
    Then 「キャンセル料率は0%から100%の範囲で入力してください」とエラーが表示される

  Scenario: キャンセル期限が未入力の場合
    Given 会議室オーナー「山田太郎」がログイン済み
    When キャンセルポリシー設定画面でキャンセル期限を空欄のまま保存を試みる
    Then 「キャンセル期限は必須です」とエラーが表示される

  Scenario: 他のオーナーの会議室のキャンセルポリシーは設定できない
    Given 会議室オーナー「山田太郎」がログイン済み
    And 会議室「新宿会議室B」はオーナー「佐藤花子」の所有
    When 会議室「新宿会議室B」のキャンセルポリシー設定画面にアクセスする
    Then 「この会議室の操作権限がありません」とエラーが表示される
```
