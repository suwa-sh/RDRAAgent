# キャンセルポリシーを設定する - フロントエンド仕様

## 変更概要

会議室オーナーがキャンセル料の発生条件や返金ルールを設定するためのキャンセルポリシー設定画面を新規作成する。

## 画面仕様

### キャンセルポリシー設定画面

- **URL**: /rooms/:room_id/cancellation-policy
- **アクセス権**: 会議室オーナー（自身の所有する会議室のみ）

#### 表示要素

| 要素 | 種別 | 説明 |
|------|------|------|
| 会議室名 | テキスト | 対象会議室の名称を表示する |
| キャンセル料率テーブル | テーブル | キャンセル期限ごとのキャンセル料率を入力する |
| キャンセル期限 | テキスト入力 | 各段階のキャンセル期限を入力する（例: 7日前まで） |
| キャンセル料率 | 数値入力 | 各段階のキャンセル料率を0-100%で入力する |
| 返金ルール | テキストエリア | 返金条件と返金方法を入力する |
| 段階追加ボタン | ボタン | キャンセル料率の段階を追加する |
| 段階削除ボタン | ボタン | キャンセル料率の段階を削除する |
| 保存ボタン | ボタン | キャンセルポリシーを保存する |
| キャンセルボタン | ボタン | 変更を破棄して会議室詳細に戻る |

#### 操作フロー

1. 会議室オーナーがキャンセルポリシー設定画面を開く
2. 既存のキャンセルポリシーがある場合は現在の値がプリセットされる
3. キャンセル期限とキャンセル料率を段階ごとに入力する
4. 必要に応じて段階を追加・削除する
5. 返金ルールを入力する
6. 保存ボタンをクリックする
7. バリデーションが成功した場合、キャンセルポリシーが保存され完了メッセージが表示される
8. バリデーションが失敗した場合、該当項目にエラーメッセージが表示される
