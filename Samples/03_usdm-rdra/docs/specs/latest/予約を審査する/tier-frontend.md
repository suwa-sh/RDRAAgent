# 予約を審査する - フロントエンド仕様

## 変更概要

会議室オーナーが予約申請一覧を確認し、利用者の評価情報を参照しながら許諾・拒否の判断を行う予約審査画面を新規作成する。

## 画面仕様

### 予約審査画面

- **URL**: /owner/reservations/review
- **アクセス権**: 会議室オーナー

#### 表示要素

| 要素 | 種別 | 説明 |
|------|------|------|
| 予約申請一覧 | テーブル | 申請状態の予約一覧（予約ID、利用者名、会議室名、予約日時、利用開始日時、利用終了日時） |
| 利用者評価サマリ | テキスト | 選択した予約の利用者の評価スコア・コメント抜粋 |
| 予約詳細 | テキスト | 予約情報の詳細（決済方法含む） |
| 許諾ボタン | ボタン | 予約を許諾する |
| 拒否ボタン | ボタン | 予約を拒否する |
| 拒否理由入力欄 | フォーム | 拒否時の理由を入力するテキストエリア |

#### 操作フロー

1. 会議室オーナーが予約審査画面を開く
2. 申請状態の予約一覧が表示される
3. 一覧から審査対象の予約を選択する
4. 予約詳細と利用者評価サマリが表示される
5. 使用許諾条件に基づき「許諾」または「拒否」を選択する
6. 拒否の場合は拒否理由を入力する
7. 確認ダイアログが表示され、確定する
