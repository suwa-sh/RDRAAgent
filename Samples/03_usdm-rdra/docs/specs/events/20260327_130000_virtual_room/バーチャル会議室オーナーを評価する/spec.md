# バーチャル会議室オーナーを評価する

## 概要

利用者がバーチャル会議室オーナーの対応を評価登録する。バーチャル会議室利用後にオーナーの対応品質を評価し、他の利用者の意思決定支援に利用される。

## 関連 RDRA モデル

| モデル種別 | 要素名 | 関連 |
|-----------|--------|------|
| BUC | 会議室評価フロー | このUCを含むBUC |
| アクター | 利用者 | 操作するアクター |
| 情報 | 会議室評価 | 登録する評価の情報 |
| バリエーション | 評価種別 | 会議室評価、オーナー評価、利用者評価 |

## E2E 完了条件（BDD）

### 正常系

```gherkin
Feature: バーチャル会議室オーナーを評価する

  Scenario: バーチャル会議室オーナーの評価を登録する
    Given 利用者「田中太郎」がバーチャル会議室「オンライン会議室A」（オーナー: 鈴木花子）を利用済みである
    When オーナー評価登録画面で評価スコア「5」、コメント「会議URL発行が迅速で丁寧な対応でした」を入力する
    And 評価登録ボタンを押下する
    Then オーナー「鈴木花子」に対するオーナー評価が登録される

  Scenario: 評価スコア3で中間評価を登録する
    Given 利用者「田中太郎」がバーチャル会議室「Teamsルーム1」（オーナー: 山田一郎）を利用済みである
    When 評価スコア「3」、コメント「普通の対応でした」を入力する
    And 評価登録ボタンを押下する
    Then オーナー「山田一郎」に対する評価スコア「3」のオーナー評価が登録される
```

### 異常系

```gherkin
  Scenario: 利用していないバーチャル会議室のオーナーは評価できない
    Given 利用者「田中太郎」がバーチャル会議室「オンライン会議室B」を利用していない
    When バーチャル会議室「オンライン会議室B」のオーナーの評価を試みる
    Then 「この会議室を利用していないためオーナーを評価できません」とエラーメッセージが表示される

  Scenario: 評価スコアが未入力の場合にエラーとなる
    Given 利用者「田中太郎」がオーナー評価登録画面を開いている
    When 評価スコアを未入力のまま評価登録ボタンを押下する
    Then 「評価スコアは必須です」とエラーメッセージが表示される
```
