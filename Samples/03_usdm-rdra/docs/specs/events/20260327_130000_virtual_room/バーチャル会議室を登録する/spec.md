# バーチャル会議室を登録する

## 概要

会議室オーナーがバーチャル会議室の会議ツール種別・同時接続数・録画可否・価格などの情報と会議URLを登録する。会議室種別別登録条件に基づき、バーチャル会議室固有の入力項目が適用される。

## 関連 RDRA モデル

| モデル種別 | 要素名 | 関連 |
|-----------|--------|------|
| BUC | 会議室管理フロー | このUCを含むBUC |
| アクター | 会議室オーナー | 操作するアクター |
| 情報 | 会議室情報 | 登録する会議室の基本情報 |
| 情報 | 会議URL | 登録する会議URLの情報 |
| 条件 | 会議室種別別登録条件 | 会議室種別に応じた登録項目切替ルール |
| バリエーション | 会議室種別 | 物理、バーチャル |
| バリエーション | 会議ツール種別 | Zoom、Teams、Google Meet |
| 状態 | 会議室 | 登録後は非公開状態 |

## E2E 完了条件（BDD）

### 正常系

```gherkin
Feature: バーチャル会議室を登録する

  Scenario: Zoomのバーチャル会議室を登録する
    Given 会議室オーナー「鈴木花子」がログイン済み
    When 会議室種別「バーチャル」を選択する
    And 会議室名「オンライン会議室A」、会議ツール種別「Zoom」、同時接続数「50」、録画可否「可」、価格「500円/時間」を入力する
    And 会議URL「https://zoom.us/j/1234567890」、有効期限「2026-12-31」を入力する
    And 登録ボタンを押下する
    Then バーチャル会議室「オンライン会議室A」が非公開状態で登録される
    And 会議URLが会議室に紐づけて保存される

  Scenario: Teamsのバーチャル会議室を登録する
    Given 会議室オーナー「鈴木花子」がログイン済み
    When 会議室種別「バーチャル」を選択する
    And 会議室名「Teamsルーム1」、会議ツール種別「Teams」、同時接続数「100」、録画可否「可」、価格「800円/時間」を入力する
    And 会議URL「https://teams.microsoft.com/l/meetup-join/xxx」、有効期限「2026-12-31」を入力する
    And 登録ボタンを押下する
    Then バーチャル会議室「Teamsルーム1」が非公開状態で登録される

  Scenario: 会議室種別をバーチャルに切り替えると入力項目が変わる
    Given 会議室オーナー「鈴木花子」がバーチャル会議室登録画面を開いている
    When 会議室種別「バーチャル」を選択する
    Then 所在地、広さ、収容人数の入力欄が非表示になる
    And 会議ツール種別、同時接続数、録画可否、会議URLの入力欄が表示される
```

### 異常系

```gherkin
  Scenario: 必須項目が未入力の場合にエラーとなる
    Given 会議室オーナー「鈴木花子」がバーチャル会議室登録画面を開いている
    When 会議室種別「バーチャル」を選択する
    And 会議室名を未入力のまま登録ボタンを押下する
    Then 「会議室名は必須です」とエラーメッセージが表示される

  Scenario: 会議URLの形式が不正な場合にエラーとなる
    Given 会議室オーナー「鈴木花子」がバーチャル会議室登録画面を開いている
    When 会議URL「invalid-url」を入力する
    And 登録ボタンを押下する
    Then 「有効な会議URLを入力してください」とエラーメッセージが表示される

  Scenario: 同時接続数が0以下の場合にエラーとなる
    Given 会議室オーナー「鈴木花子」がバーチャル会議室登録画面を開いている
    When 同時接続数「0」を入力する
    And 登録ボタンを押下する
    Then 「同時接続数は1以上を指定してください」とエラーメッセージが表示される
```
