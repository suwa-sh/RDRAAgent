# バーチャル会議室を予約する

## 概要

利用者がバーチャル会議室の利用日時を指定して予約申請を行う。決済フローは物理会議室と同じフローを使用する。予約情報には会議室種別が含まれ、バーチャル会議室であることが識別される。

## 関連 RDRA モデル

| モデル種別 | 要素名 | 関連 |
|-----------|--------|------|
| BUC | 会議室予約フロー | このUCを含むBUC |
| アクター | 利用者 | 操作するアクター |
| 情報 | 予約情報 | 登録する予約の情報 |
| 情報 | 会議室情報 | 予約対象の会議室情報 |
| 状態 | 予約 | 予約申請により「申請」状態に遷移 |
| バリエーション | 会議室種別 | 物理、バーチャル |

## E2E 完了条件（BDD）

### 正常系

```gherkin
Feature: バーチャル会議室を予約する

  Scenario: Zoomのバーチャル会議室を予約する
    Given 利用者「田中太郎」がログイン済み
    And 公開中のバーチャル会議室「オンライン会議室A」が登録されている（会議ツール種別: Zoom、同時接続数: 50）
    When バーチャル会議室「オンライン会議室A」の予約画面を開く
    And 利用日時「2026-04-01 10:00〜12:00」を指定する
    And 予約申請ボタンを押下する
    Then 予約が「申請」状態で登録される
    And 予約情報の会議室種別が「バーチャル」で保存される

  Scenario: バーチャル会議室の予約で決済方法を設定する
    Given 利用者「田中太郎」がバーチャル会議室の予約申請を完了している
    When 決済方法「クレジットカード」を選択して設定する
    Then 決済情報が予約に紐づけて保存される
```

### 異常系

```gherkin
  Scenario: 利用日時が過去の場合にエラーとなる
    Given 利用者「田中太郎」がバーチャル会議室予約画面を開いている
    When 利用日時「2025-01-01 10:00〜12:00」を指定する
    And 予約申請ボタンを押下する
    Then 「過去の日時は指定できません」とエラーメッセージが表示される

  Scenario: 非公開のバーチャル会議室は予約できない
    Given バーチャル会議室「非公開オンラインルーム」が非公開状態で登録されている
    When 利用者「田中太郎」が「非公開オンラインルーム」の予約を試みる
    Then 予約画面にアクセスできず、エラーが表示される

  Scenario: 既に予約済みの時間帯には予約できない
    Given バーチャル会議室「オンライン会議室A」に「2026-04-01 10:00〜12:00」の予約が存在する
    When 利用者「田中太郎」が同じ時間帯「2026-04-01 10:00〜12:00」で予約申請する
    Then 「指定された時間帯は既に予約されています」とエラーメッセージが表示される
```
