# 運用ルールを設定する

## 概要

会議室オーナーが会議室の利用可能時間帯、最低利用時間、最大利用時間、貸出可否などの運用ルールを設定するユースケース。運用ルールは会議室公開条件の一部であり、公開可能にするために設定が必要。

## 関連 RDRA モデル

| モデル種別 | 要素名 | 関連 |
|-----------|--------|------|
| BUC | 会議室管理フロー | このUCを含むBUC |
| アクター | 会議室オーナー | 操作するアクター |
| 情報 | 運用ルール | 設定する情報 |

## E2E 完了条件（BDD）

### 正常系

```gherkin
Feature: 運用ルールを設定する

  Scenario: 会議室オーナーが運用ルールを新規設定する
    Given 会議室オーナー「山田太郎」がログイン済み
    And 会議室「渋谷ミーティングルームA」が登録済み
    And 運用ルールが未設定
    When 運用ルール設定画面で以下の情報を入力して保存する
      | 項目           | 値          |
      | 利用可能時間帯 | 9:00-21:00  |
      | 最低利用時間   | 1時間       |
      | 最大利用時間   | 8時間       |
      | 貸出可否       | 可          |
    Then 運用ルールが設定される

  Scenario: 会議室オーナーが運用ルールを変更する
    Given 会議室オーナー「山田太郎」がログイン済み
    And 会議室「渋谷ミーティングルームA」に運用ルールが設定済み
    When 運用ルール設定画面で利用可能時間帯を「8:00-22:00」に変更して保存する
    Then 利用可能時間帯が「8:00-22:00」に更新される
```

### 異常系

```gherkin
  Scenario: 利用可能時間帯に不正な値を入力した場合
    Given 会議室オーナー「山田太郎」がログイン済み
    When 運用ルール設定画面で利用可能時間帯に「25:00-30:00」を入力して保存を試みる
    Then 「正しい時間帯を入力してください」とエラーが表示される

  Scenario: 最低利用時間が最大利用時間を超える場合
    Given 会議室オーナー「山田太郎」がログイン済み
    When 運用ルール設定画面で最低利用時間「10時間」、最大利用時間「8時間」を入力して保存を試みる
    Then 「最低利用時間は最大利用時間以下にしてください」とエラーが表示される

  Scenario: 他のオーナーの会議室の運用ルールは設定できない
    Given 会議室オーナー「山田太郎」がログイン済み
    And 会議室「新宿会議室B」はオーナー「佐藤花子」の所有
    When 会議室「新宿会議室B」の運用ルール設定画面にアクセスする
    Then 「この会議室の操作権限がありません」とエラーが表示される
```
