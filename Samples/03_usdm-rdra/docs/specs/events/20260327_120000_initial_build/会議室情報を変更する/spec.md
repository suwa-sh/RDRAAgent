# 会議室情報を変更する

## 概要

会議室オーナーが登録済みの会議室情報（会議室名、所在地、広さ、収容人数、価格、設備・機能、画像）を修正・更新するユースケース。公開状態に関わらず情報の変更が可能。

## 関連 RDRA モデル

| モデル種別 | 要素名 | 関連 |
|-----------|--------|------|
| BUC | 会議室管理フロー | このUCを含むBUC |
| アクター | 会議室オーナー | 操作するアクター |
| 情報 | 会議室情報 | 変更対象の情報 |

## E2E 完了条件（BDD）

### 正常系

```gherkin
Feature: 会議室情報を変更する

  Scenario: 会議室オーナーが会議室名を変更する
    Given 会議室オーナー「山田太郎」がログイン済み
    And 会議室「渋谷ミーティングルームA」が登録済み
    When 会議室情報編集画面で会議室名を「渋谷ミーティングルームA プレミアム」に変更して保存する
    Then 会議室名が「渋谷ミーティングルームA プレミアム」に更新される

  Scenario: 会議室の設備情報を変更する
    Given 会議室オーナー「山田太郎」がログイン済み
    And 会議室「渋谷ミーティングルームA」に設備「プロジェクター, Wi-Fi」が登録済み
    When 会議室情報編集画面で設備に「テレビ会議設備」を追加して保存する
    Then 会議室の設備が「プロジェクター, Wi-Fi, テレビ会議設備」に更新される

  Scenario: 会議室の価格を変更する
    Given 会議室オーナー「山田太郎」がログイン済み
    And 会議室「渋谷ミーティングルームA」の価格が「3000円/時間」
    When 会議室情報編集画面で価格を「3500円/時間」に変更して保存する
    Then 会議室の価格が「3500円/時間」に更新される
```

### 異常系

```gherkin
  Scenario: 他のオーナーの会議室情報は変更できない
    Given 会議室オーナー「山田太郎」がログイン済み
    And 会議室「新宿会議室B」はオーナー「佐藤花子」の所有
    When 会議室「新宿会議室B」の情報編集画面にアクセスする
    Then 「この会議室の編集権限がありません」とエラーが表示される

  Scenario: 必須項目を空にして保存できない
    Given 会議室オーナー「山田太郎」がログイン済み
    And 会議室「渋谷ミーティングルームA」の情報編集画面を開いている
    When 会議室名を空欄にして保存を試みる
    Then 「会議室名は必須です」とエラーが表示される
```
