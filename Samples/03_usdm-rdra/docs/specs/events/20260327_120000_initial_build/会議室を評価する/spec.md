# 会議室を評価する

## 概要

利用者が利用した会議室の設備・清潔さなどを評価登録する。会議室評価情報が作成され、他の利用者の会議室選択の参考情報として活用される。

## 関連 RDRA モデル

| モデル種別 | 要素名 | 関連 |
|-----------|--------|------|
| BUC | 会議室評価フロー | このUCを含むBUC |
| アクター | 利用者 | 操作するアクター |
| 情報 | 会議室評価 | 作成する評価情報 |
| バリエーション | 評価種別 | 会議室評価 |

## E2E 完了条件（BDD）

### 正常系

```gherkin
Feature: 会議室を評価する

  Scenario: 利用した会議室を評価する
    Given 利用者「田中太郎」がログイン済み
    And 会議室「新宿カンファレンスルームA」の利用が完了している（利用終了済み）
    When 評価スコア「4」、コメント「設備が充実しており快適でした」で評価を登録する
    Then 会議室評価が作成される
    And 評価種別が「会議室評価」である
    And 評価登録完了メッセージが表示される

  Scenario: 評価スコアのみで評価する（コメントなし）
    Given 利用者「田中太郎」がログイン済み
    And 会議室「新宿カンファレンスルームA」の利用が完了している
    When 評価スコア「5」のみで評価を登録する
    Then 会議室評価が作成される
```

### 異常系

```gherkin
  Scenario: 利用していない会議室を評価しようとする
    Given 利用者「田中太郎」がログイン済み
    And 会議室「渋谷ミーティングスペースB」を利用したことがない
    When 会議室「渋谷ミーティングスペースB」を評価しようとする
    Then 「利用済みの会議室のみ評価できます」とエラーが表示される

  Scenario: 評価スコアが範囲外の場合
    Given 利用者「田中太郎」がログイン済み
    And 会議室「新宿カンファレンスルームA」の利用が完了している
    When 評価スコア「6」で評価を登録しようとする
    Then 「評価スコアは1〜5の範囲で入力してください」とエラーが表示される
```
