# オーナー情報を変更する

## 概要

会議室オーナーが自身の登録情報（氏名、連絡先、メールアドレス等）を変更する。登録済み状態のオーナーのみが情報変更可能である。

## 関連 RDRA モデル

| モデル種別 | 要素名 | 関連 |
|-----------|--------|------|
| BUC | オーナー登録管理フロー | このUCを含むBUC |
| アクター | 会議室オーナー | 操作するアクター |
| 情報 | オーナー情報 | 参照・更新する情報 |

## E2E 完了条件（BDD）

### 正常系

```gherkin
Feature: オーナー情報を変更する

  Scenario: オーナーが氏名を変更する
    Given 会議室オーナー「鈴木一郎」がオーナー情報編集画面を表示している
    And オーナー「鈴木一郎」の審査状態が「登録済み」である
    When 氏名を「鈴木太郎」に変更して保存ボタンを押す
    Then オーナー情報の氏名が「鈴木太郎」に更新される
    And 変更完了メッセージが表示される

  Scenario: オーナーが連絡先とメールアドレスを変更する
    Given 会議室オーナー「鈴木一郎」がオーナー情報編集画面を表示している
    And オーナー「鈴木一郎」の審査状態が「登録済み」である
    When 連絡先を「090-1111-2222」、メールアドレスを「new-suzuki@example.com」に変更して保存ボタンを押す
    Then オーナー情報の連絡先が「090-1111-2222」、メールアドレスが「new-suzuki@example.com」に更新される
```

### 異常系

```gherkin
  Scenario: 必須項目を空にして保存した場合にエラーとなる
    Given 会議室オーナー「鈴木一郎」がオーナー情報編集画面を表示している
    When 氏名を空にして保存ボタンを押す
    Then 「氏名は必須です」とバリデーションエラーが表示される
    And オーナー情報は更新されない

  Scenario: メールアドレスの形式が不正な場合にエラーとなる
    Given 会議室オーナー「鈴木一郎」がオーナー情報編集画面を表示している
    When メールアドレスを「invalid-email」に変更して保存ボタンを押す
    Then 「メールアドレスの形式が正しくありません」とバリデーションエラーが表示される
    And オーナー情報は更新されない

  Scenario: 他のオーナーが使用中のメールアドレスに変更した場合にエラーとなる
    Given 会議室オーナー「鈴木一郎」がオーナー情報編集画面を表示している
    And メールアドレス「tanaka@example.com」が別のオーナーで登録済みである
    When メールアドレスを「tanaka@example.com」に変更して保存ボタンを押す
    Then 「このメールアドレスは既に使用されています」とエラーが表示される
    And オーナー情報は更新されない

  Scenario: 登録済み以外のオーナーが情報変更を試みた場合にエラーとなる
    Given 会議室オーナー「田中二郎」の審査状態が「審査待ち」である
    When オーナー情報編集画面にアクセスしようとする
    Then 「登録済みのオーナーのみ情報変更が可能です」とエラーが表示される
```
