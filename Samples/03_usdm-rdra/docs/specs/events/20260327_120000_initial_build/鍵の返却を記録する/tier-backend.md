# 鍵の返却を記録する - バックエンド仕様

## 変更概要

鍵の返却記録を行い、鍵の状態を「貸出中」から「保管中」に、会議室利用を「利用終了」状態に遷移させるAPIを新規作成する。

## API 仕様

### 貸出中鍵一覧取得API

- **メソッド**: GET
- **パス**: /api/owner/keys/lent
- **認証**: 会議室オーナー認証

#### リクエスト

| パラメータ | 型 | 必須 | 説明 |
|-----------|---|------|------|
| room_id | string | No | 会議室IDで絞り込み |

#### レスポンス

| フィールド | 型 | 説明 |
|-----------|---|------|
| keys | array | 貸出中の鍵一覧 |
| keys[].key_id | string | 鍵ID |
| keys[].room_name | string | 会議室名 |
| keys[].user_name | string | 利用者名 |
| keys[].lent_at | string | 貸出日時 |
| keys[].usage_id | string | 会議室利用ID |
| keys[].usage_status | string | 会議室利用の状態 |

### 鍵返却記録API

- **メソッド**: POST
- **パス**: /api/owner/keys/{key_id}/return
- **認証**: 会議室オーナー認証

#### リクエスト

| パラメータ | 型 | 必須 | 説明 |
|-----------|---|------|------|
| key_id | string | Yes | 鍵ID（パスパラメータ） |

#### レスポンス

| フィールド | 型 | 説明 |
|-----------|---|------|
| key_id | string | 鍵ID |
| key_status | string | 遷移後の鍵状態（保管中） |
| usage_id | string | 会議室利用ID |
| usage_status | string | 会議室利用の状態（利用終了） |
| returned_at | string | 返却日時 |

## データモデル変更

### 鍵

| カラム | 型 | 説明 | 変更種別 |
|--------|---|------|---------|
| 鍵ID | VARCHAR | 鍵の一意識別子 | 追加 |
| 会議室ID | VARCHAR | 対象会議室のID | 追加 |
| 貸出状態 | VARCHAR | 保管中/貸出中 | 追加 |
| 貸出日時 | DATETIME | 貸出記録日時 | 追加 |
| 返却日時 | DATETIME | 返却記録日時 | 追加 |

### 会議室利用

| カラム | 型 | 説明 | 変更種別 |
|--------|---|------|---------|
| 利用ID | VARCHAR | 利用の一意識別子 | 追加 |
| 予約ID | VARCHAR | 対応する予約ID | 追加 |
| 会議室ID | VARCHAR | 利用する会議室ID | 追加 |
| 利用者ID | VARCHAR | 利用者のID | 追加 |
| 利用開始日時 | DATETIME | 利用開始日時 | 追加 |
| 利用終了日時 | DATETIME | 利用終了日時 | 追加 |
| 利用状態 | VARCHAR | 利用開始/利用中/利用終了 | 追加 |

## ビジネスルール

- 会議室利用ポリシー: 鍵の返却をトリガーとして会議室利用の終了を定義する
- 鍵の状態が「貸出中」の場合のみ返却可能
- 返却記録時に鍵の状態を「貸出中」→「保管中」に遷移
- 同時に会議室利用の状態を「利用中」→「利用終了」に遷移
- 利用終了日時に返却日時を記録

## ティア完了条件（BDD）

```gherkin
Feature: 鍵の返却を記録する - バックエンド

  Scenario: 鍵を返却し会議室利用を終了する
    Given 鍵ID「KEY-001」が「貸出中」状態である
    And 対応する会議室利用が「利用中」状態である
    When POST /api/owner/keys/KEY-001/return を実行する
    Then ステータスコード200が返却される
    And 鍵状態が「保管中」に更新される
    And 会議室利用が「利用終了」状態に更新される
    And 返却日時が記録される

  Scenario: 保管中の鍵を返却しようとする
    Given 鍵ID「KEY-002」が「保管中」状態である
    When POST /api/owner/keys/KEY-002/return を実行する
    Then ステータスコード409が返却される
    And エラーメッセージ「この鍵は貸出中ではありません」が返却される
```
