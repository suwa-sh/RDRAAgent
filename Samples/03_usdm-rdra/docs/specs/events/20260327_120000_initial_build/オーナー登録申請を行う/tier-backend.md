# オーナー登録申請を行う - バックエンド仕様

## 変更概要

オーナー登録申請を受け付け、オーナー情報を新規作成して審査状態を「審査待ち」に設定するAPIを新規作成する。

## API 仕様

### オーナー登録申請API

- **メソッド**: POST
- **パス**: /api/owners/register
- **認証**: なし（未認証ユーザーからの申請）

#### リクエスト

| パラメータ | 型 | 必須 | 説明 |
|-----------|---|------|------|
| name | string | Yes | オーナーの氏名 |
| contact | string | Yes | オーナーの連絡先（電話番号） |
| email | string | Yes | オーナーのメールアドレス |

#### レスポンス

| フィールド | 型 | 説明 |
|-----------|---|------|
| ownerId | string | 採番されたオーナーID |
| name | string | オーナーの氏名 |
| email | string | オーナーのメールアドレス |
| registrationDate | string | 登録日（ISO 8601形式） |
| reviewStatus | string | 審査状態（「審査待ち」） |

## データモデル変更

### オーナー情報

| カラム | 型 | 説明 | 変更種別 |
|--------|---|------|---------|
| オーナーID | VARCHAR | オーナーの一意識別子 | 追加 |
| 氏名 | VARCHAR | オーナーの氏名 | 追加 |
| 連絡先 | VARCHAR | オーナーの電話番号 | 追加 |
| メールアドレス | VARCHAR | オーナーのメールアドレス | 追加 |
| 登録日 | DATE | 登録申請日 | 追加 |
| 審査状態 | VARCHAR | オーナーの審査状態 | 追加 |
| 退会日 | DATE | 退会日（NULL許容） | 追加 |

## ビジネスルール

- オーナーIDはシステムで自動採番する
- 登録日はAPI実行時の日付を設定する
- 審査状態の初期値は「審査待ち」とする
- メールアドレスはオーナー情報テーブル内で一意でなければならない
- 氏名、連絡先、メールアドレスは全て必須項目とする
