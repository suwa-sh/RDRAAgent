# オーナー登録申請を行う - フロントエンド仕様

## 変更概要

会議室オーナーがサービスへの登録申請情報を入力・送信するためのオーナー登録申請画面を新規作成する。

## 画面仕様

### オーナー登録申請画面

- **URL**: /owners/register
- **アクセス権**: 未認証ユーザー（会議室オーナー）

#### 表示要素

| 要素 | 種別 | 説明 |
|------|------|------|
| 氏名 | テキスト入力 | オーナーの氏名を入力する。必須項目 |
| 連絡先 | テキスト入力 | オーナーの電話番号を入力する。必須項目 |
| メールアドレス | テキスト入力 | オーナーのメールアドレスを入力する。必須項目 |
| 申請ボタン | ボタン | 入力内容を送信しオーナー登録申請を実行する |
| 申請完了メッセージ | テキスト | 申請成功時に表示される完了メッセージ |
| バリデーションエラー | テキスト | 入力不備時に各項目付近に表示されるエラーメッセージ |

#### 操作フロー

1. 会議室オーナーがオーナー登録申請画面にアクセスする
2. 氏名、連絡先、メールアドレスを入力する
3. 申請ボタンを押す
4. 入力内容のバリデーションが実行される
5. バリデーション成功時、APIにリクエストが送信される
6. 申請完了メッセージが表示される
