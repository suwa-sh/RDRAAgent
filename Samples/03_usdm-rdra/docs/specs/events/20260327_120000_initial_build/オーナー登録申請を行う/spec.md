# オーナー登録申請を行う

## 概要

会議室オーナーがサービスへの登録申請情報を入力し、オーナー登録申請を行う。申請が完了するとオーナーの状態が「審査待ち」に遷移する。

## 関連 RDRA モデル

| モデル種別 | 要素名 | 関連 |
|-----------|--------|------|
| BUC | オーナー登録管理フロー | このUCを含むBUC |
| アクター | 会議室オーナー | 操作するアクター |
| 情報 | オーナー情報 | 登録・更新する情報 |
| 状態 | オーナー | 「（初期）→ 審査待ち」に遷移 |

## E2E 完了条件（BDD）

### 正常系

```gherkin
Feature: オーナー登録申請を行う

  Scenario: 新規オーナーが登録申請を完了する
    Given 会議室オーナー「鈴木一郎」がオーナー登録申請画面を表示している
    When 氏名「鈴木一郎」、連絡先「03-1234-5678」、メールアドレス「suzuki@example.com」を入力して申請ボタンを押す
    Then オーナー情報が新規作成され審査状態が「審査待ち」になる
    And 申請完了メッセージが表示される

  Scenario: 全ての必須項目を入力して申請する
    Given 会議室オーナー「佐藤花子」がオーナー登録申請画面を表示している
    When 氏名「佐藤花子」、連絡先「090-9876-5432」、メールアドレス「sato@example.com」を入力して申請ボタンを押す
    Then オーナー情報にオーナーIDが採番され登録日が本日日付で設定される
    And 審査状態が「審査待ち」になる
```

### 異常系

```gherkin
  Scenario: 必須項目が未入力の場合にエラーとなる
    Given 会議室オーナー「鈴木一郎」がオーナー登録申請画面を表示している
    When 氏名を空欄のまま申請ボタンを押す
    Then 「氏名は必須です」とバリデーションエラーが表示される
    And オーナー情報は作成されない

  Scenario: メールアドレスの形式が不正な場合にエラーとなる
    Given 会議室オーナー「鈴木一郎」がオーナー登録申請画面を表示している
    When メールアドレス「invalid-email」を入力して申請ボタンを押す
    Then 「メールアドレスの形式が正しくありません」とバリデーションエラーが表示される
    And オーナー情報は作成されない

  Scenario: 既に登録済みのメールアドレスで申請した場合にエラーとなる
    Given メールアドレス「suzuki@example.com」で既にオーナー登録が存在する
    When 会議室オーナー「鈴木二郎」がメールアドレス「suzuki@example.com」で申請ボタンを押す
    Then 「このメールアドレスは既に登録されています」とエラーが表示される
    And オーナー情報は作成されない
```
