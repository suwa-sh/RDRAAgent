# 鍵の貸出を記録する

## 概要

会議室オーナーが利用者に鍵を渡し、鍵の状態を「保管中」から「貸出中」に遷移させるとともに、会議室利用を「利用開始」状態に遷移させるユースケース。会議室利用ポリシーに基づき、鍵の受け渡しをトリガーとして利用開始を記録する。

## 関連 RDRA モデル

| モデル種別 | 要素名 | 関連 |
|-----------|--------|------|
| BUC | 会議室貸出管理フロー | このUCを含むBUC |
| アクター | 会議室オーナー | 操作するアクター |
| 情報 | 鍵 | 更新する鍵の貸出状態 |
| 情報 | 会議室利用 | 更新する会議室利用の状態 |
| 状態 | 鍵 | 保管中→貸出中の状態遷移 |
| 状態 | 会議室利用 | →利用開始の状態遷移 |
| 条件 | 会議室利用ポリシー | 鍵の受け渡しによる利用開始・終了のルール |

## E2E 完了条件（BDD）

### 正常系

```gherkin
Feature: 鍵の貸出を記録する

  Scenario: 鍵を貸出し会議室利用を開始する
    Given 会議室オーナー「山田花子」がログイン済み
    And 会議室「新宿会議室A」の鍵が「保管中」状態である
    And 利用者「田中太郎」の予約ID「RSV-001」が「確定」状態である
    When 会議室オーナー「山田花子」が鍵貸出記録画面で利用者「田中太郎」への鍵貸出を記録する
    Then 鍵の状態が「貸出中」に遷移する
    And 会議室利用が「利用開始」状態で作成される
    And 貸出日時が記録される
```

### 異常系

```gherkin
  Scenario: 既に貸出中の鍵を貸出しようとする
    Given 会議室オーナー「山田花子」がログイン済み
    And 会議室「新宿会議室A」の鍵が「貸出中」状態である
    When 鍵の貸出を記録しようとする
    Then エラーメッセージ「この鍵は既に貸出中です」が表示される

  Scenario: 確定済みでない予約に対して鍵を貸出しようとする
    Given 会議室オーナー「山田花子」がログイン済み
    And 利用者「田中太郎」の予約が「申請」状態である
    When 鍵の貸出を記録しようとする
    Then エラーメッセージ「予約が確定されていません」が表示される
```
