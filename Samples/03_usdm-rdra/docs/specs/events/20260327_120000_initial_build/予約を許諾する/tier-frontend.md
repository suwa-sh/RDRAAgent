# 予約を許諾する - フロントエンド仕様

## 変更概要

会議室オーナーが予約申請・変更申請を確認し、許諾を実行する予約許諾画面を新規作成する。

## 画面仕様

### 予約許諾画面

- **URL**: /owner/reservations/approve
- **アクセス権**: 会議室オーナー

#### 表示要素

| 要素 | 種別 | 説明 |
|------|------|------|
| 許諾待ち予約一覧 | テーブル | 申請・変更状態の予約一覧（予約ID、利用者名、会議室名、利用開始日時、利用終了日時、予約状態） |
| 予約詳細 | テキスト | 選択した予約の詳細情報 |
| 利用者情報 | テキスト | 予約した利用者の基本情報と評価スコア |
| 許諾ボタン | ボタン | 予約を許諾し確定状態に遷移させる |
| 確認ダイアログ | テキスト | 許諾実行前の確認メッセージ |

#### 操作フロー

1. 会議室オーナーが予約許諾画面を開く
2. 申請・変更状態の予約が一覧表示される
3. 許諾対象の予約を選択する
4. 予約詳細と利用者情報が表示される
5. 「許諾」ボタンを押下する
6. 確認ダイアログで「確定」を押下する
7. 予約が確定状態に遷移し、一覧が更新される
