# 予約を許諾する

## 概要

会議室オーナーが利用者の予約申請を確認し、使用許諾条件に基づいて予約を確定状態に遷移させるユースケース。予約状態を「申請」または「変更」から「確定」に遷移させる。

## 関連 RDRA モデル

| モデル種別 | 要素名 | 関連 |
|-----------|--------|------|
| BUC | 会議室予約フロー | このUCを含むBUC |
| アクター | 会議室オーナー | 操作するアクター |
| 情報 | 予約情報 | 更新する予約の状態 |
| 状態 | 予約 | 申請→確定、変更→確定の状態遷移 |
| 条件 | 使用許諾条件 | 許諾判断に適用される条件 |
| バリエーション | 評価種別 | 許諾基準に関連する評価種別 |

## E2E 完了条件（BDD）

### 正常系

```gherkin
Feature: 予約を許諾する

  Scenario: 申請状態の予約を許諾し確定にする
    Given 会議室オーナー「山田花子」がログイン済み
    And 利用者「田中太郎」から会議室「新宿会議室A」への予約が「申請」状態である
    When 会議室オーナー「山田花子」が予約許諾画面で「許諾」を実行する
    Then 予約状態が「確定」に遷移する
    And 利用者「田中太郎」に予約確定の通知が送信される

  Scenario: 変更状態の予約を許諾し確定にする
    Given 会議室オーナー「山田花子」がログイン済み
    And 利用者「田中太郎」の予約が「変更」状態である
    When 会議室オーナー「山田花子」が予約許諾画面で「許諾」を実行する
    Then 予約状態が「確定」に遷移する
```

### 異常系

```gherkin
  Scenario: 取消済みの予約を許諾しようとする
    Given 会議室オーナー「山田花子」がログイン済み
    And 利用者「田中太郎」の予約が「取消」状態である
    When 会議室オーナー「山田花子」が当該予約の許諾を試みる
    Then エラーメッセージ「取消済みの予約は許諾できません」が表示される

  Scenario: 既に確定済みの予約を許諾しようとする
    Given 会議室オーナー「山田花子」がログイン済み
    And 利用者「田中太郎」の予約が既に「確定」状態である
    When 会議室オーナー「山田花子」が当該予約の許諾を試みる
    Then エラーメッセージ「この予約は既に確定済みです」が表示される
```
