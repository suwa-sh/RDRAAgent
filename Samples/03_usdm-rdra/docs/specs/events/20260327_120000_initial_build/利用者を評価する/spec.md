# 利用者を評価する

## 概要

会議室オーナーが利用者の利用マナーなどを評価登録するユースケース。評価結果は次回以降の使用許諾判断の参考情報として利用される。

## 関連 RDRA モデル

| モデル種別 | 要素名 | 関連 |
|-----------|--------|------|
| BUC | 会議室貸出管理フロー | このUCを含むBUC |
| アクター | 会議室オーナー | 操作するアクター |
| 情報 | 利用者評価 | 登録する利用者の評価情報 |
| バリエーション | 評価種別 | 評価の区分（利用者評価） |

## E2E 完了条件（BDD）

### 正常系

```gherkin
Feature: 利用者を評価する

  Scenario: 利用者の利用マナーを評価登録する
    Given 会議室オーナー「山田花子」がログイン済み
    And 利用者「田中太郎」の会議室利用が「利用終了」状態である
    When 会議室オーナー「山田花子」が利用者評価登録画面で評価スコア「4」、コメント「マナー良好、時間厳守でした」を入力する
    And 「登録」ボタンを押下する
    Then 利用者評価が登録される
    And 評価日時が記録される

  Scenario: 高評価を登録する
    Given 会議室オーナー「山田花子」がログイン済み
    And 利用者「鈴木一郎」の会議室利用が「利用終了」状態である
    When 評価スコア「5」、コメント「非常に丁寧にご利用いただきました」を入力し登録する
    Then 利用者評価が登録される
```

### 異常系

```gherkin
  Scenario: 評価スコアを入力せずに登録しようとする
    Given 会議室オーナー「山田花子」がログイン済み
    When 評価スコアを未入力のまま「登録」ボタンを押下する
    Then エラーメッセージ「評価スコアは必須です」が表示される

  Scenario: 利用終了していない利用者を評価しようとする
    Given 会議室オーナー「山田花子」がログイン済み
    And 利用者「田中太郎」の会議室利用が「利用中」状態である
    When 利用者「田中太郎」の評価登録を試みる
    Then エラーメッセージ「利用終了後に評価してください」が表示される
```
