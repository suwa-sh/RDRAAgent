# サービスへ問合せする

## 概要

利用者がサービスに関する問合せを送信する。問合せ内容はサービス運営担当者宛に登録され、未対応状態として管理される。

## 関連 RDRA モデル

| モデル種別 | 要素名 | 関連 |
|-----------|--------|------|
| BUC | 問合せ管理フロー | このUCを含むBUC |
| アクター | 利用者 | 操作するアクター |
| 情報 | 問合せ | 参照・更新する情報（問合せ内容の登録） |
| 状態 | 問合せ | 未対応状態への遷移（問合せ種別：サービス運営宛問合せ） |
| バリエーション | 問合せ種別 | サービス運営宛問合せ |

## E2E 完了条件（BDD）

### 正常系

```gherkin
Feature: サービスへ問合せする

  Scenario: サービスへの問合せを送信する
    Given 利用者「田中太郎」がログイン済み
    When サービス問合せ画面で問合せ内容「予約のキャンセルについて教えてください」を入力し送信する
    Then 問合せが登録される
    And 問合せ状態が「未対応」となる
    And 問合せ先区分が「サービス運営宛問合せ」として記録される
    And 「問合せを送信しました」と完了メッセージが表示される

  Scenario: 問合せ送信後に問合せ履歴を確認する
    Given 利用者「田中太郎」がログイン済み
    And 過去に問合せ「Q001」を送信済みである
    When サービス問合せ画面で問合せ履歴を表示する
    Then 問合せ「Q001」の問合せ内容・問合せ状態・問合せ日時が表示される
```

### 異常系

```gherkin
  Scenario: 問合せ内容が空の場合
    Given 利用者「田中太郎」がログイン済み
    When サービス問合せ画面で問合せ内容を空のまま送信する
    Then 「問合せ内容を入力してください」とバリデーションエラーが表示される

  Scenario: 未ログイン状態で問合せしようとした場合
    Given 未ログイン状態である
    When サービス問合せ画面にアクセスする
    Then ログイン画面にリダイレクトされる
```
