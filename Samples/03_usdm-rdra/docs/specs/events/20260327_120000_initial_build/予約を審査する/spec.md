# 予約を審査する

## 概要

会議室オーナーが利用者からの予約申請内容と利用者の過去の利用評価を確認し、使用許諾条件に基づいて予約の許諾または拒否を判断するユースケース。

## 関連 RDRA モデル

| モデル種別 | 要素名 | 関連 |
|-----------|--------|------|
| BUC | 会議室貸出管理フロー | このUCを含むBUC |
| アクター | 会議室オーナー | 操作するアクター |
| 情報 | 予約情報 | 参照する予約申請内容 |
| 情報 | 利用者評価 | 参照する利用者の過去評価 |
| 条件 | 使用許諾条件 | 許諾判断に適用される条件 |
| バリエーション | 評価種別 | 評価種別ごとの許諾基準 |

## E2E 完了条件（BDD）

### 正常系

```gherkin
Feature: 予約を審査する

  Scenario: オーナーが利用者評価を確認し予約を許諾する
    Given 会議室オーナー「山田花子」がログイン済み
    And 利用者「田中太郎」から会議室「新宿会議室A」への予約申請が存在する
    And 利用者「田中太郎」の利用者評価スコアが「4.5」である
    When 会議室オーナー「山田花子」が予約審査画面で予約申請を確認する
    And 利用者評価を確認し「許諾」を選択する
    Then 予約状態が「申請」のまま許諾待ち処理が完了する
    And 利用者「田中太郎」に許諾判断の結果が通知される

  Scenario: オーナーが予約を拒否する
    Given 会議室オーナー「山田花子」がログイン済み
    And 利用者「佐藤次郎」から会議室「新宿会議室A」への予約申請が存在する
    And 利用者「佐藤次郎」の利用者評価スコアが「2.0」である
    When 会議室オーナー「山田花子」が予約審査画面で予約申請を確認する
    And 利用者評価を確認し「拒否」を選択する
    Then 予約申請が拒否される
    And 利用者「佐藤次郎」に拒否の結果が通知される
```

### 異常系

```gherkin
  Scenario: 既に取り消された予約を審査しようとする
    Given 会議室オーナー「山田花子」がログイン済み
    And 利用者「田中太郎」の予約が既に「取消」状態である
    When 会議室オーナー「山田花子」が当該予約の審査を試みる
    Then エラーメッセージ「この予約は既に取り消されています」が表示される

  Scenario: ログインしていないオーナーが審査画面にアクセスする
    Given 未ログイン状態である
    When 予約審査画面にアクセスする
    Then ログイン画面にリダイレクトされる
```
