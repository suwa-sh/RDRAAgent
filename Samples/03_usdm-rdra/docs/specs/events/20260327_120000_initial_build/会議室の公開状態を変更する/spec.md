# 会議室の公開状態を変更する

## 概要

会議室オーナーが会議室の公開状態（非公開、公開可能、公開中）を切り替えるユースケース。会議室公開条件を満たした場合に公開可能となり、公開中にすることで利用者が検索・予約できるようになる。

## 関連 RDRA モデル

| モデル種別 | 要素名 | 関連 |
|-----------|--------|------|
| BUC | 会議室管理フロー | このUCを含むBUC |
| アクター | 会議室オーナー | 操作するアクター |
| 情報 | 会議室情報 | 公開状態を持つ情報 |
| 状態 | 会議室 | 非公開 → 公開可能 → 公開中の状態遷移 |
| 条件 | 会議室公開条件 | 公開可能にするための条件 |

## E2E 完了条件（BDD）

### 正常系

```gherkin
Feature: 会議室の公開状態を変更する

  Scenario: 非公開の会議室を公開可能にする
    Given 会議室オーナー「山田太郎」がログイン済み
    And 会議室「渋谷ミーティングルームA」が「非公開」状態
    And 会議室の物件情報と運用ルールとキャンセルポリシーが登録済み
    When 会議室公開管理画面で「公開可能」に変更する
    Then 会議室の公開状態が「公開可能」に変更される

  Scenario: 公開可能の会議室を公開中にする
    Given 会議室オーナー「山田太郎」がログイン済み
    And 会議室「渋谷ミーティングルームA」が「公開可能」状態
    When 会議室公開管理画面で「公開中」に変更する
    Then 会議室の公開状態が「公開中」に変更される
    And 利用者が会議室を検索できるようになる

  Scenario: 公開中の会議室を非公開にする
    Given 会議室オーナー「山田太郎」がログイン済み
    And 会議室「渋谷ミーティングルームA」が「公開中」状態
    When 会議室公開管理画面で「非公開」に変更する
    Then 会議室の公開状態が「非公開」に変更される
    And 利用者が会議室を検索できなくなる

  Scenario: 公開可能の会議室を非公開にする
    Given 会議室オーナー「山田太郎」がログイン済み
    And 会議室「渋谷ミーティングルームA」が「公開可能」状態
    When 会議室公開管理画面で「非公開」に変更する
    Then 会議室の公開状態が「非公開」に変更される
```

### 異常系

```gherkin
  Scenario: 会議室公開条件を満たさない場合に公開可能にできない
    Given 会議室オーナー「山田太郎」がログイン済み
    And 会議室「渋谷ミーティングルームA」が「非公開」状態
    And 運用ルールが未設定
    When 会議室公開管理画面で「公開可能」に変更を試みる
    Then 「運用ルールが未設定のため公開できません」とエラーが表示される

  Scenario: キャンセルポリシー未設定で公開可能にできない
    Given 会議室オーナー「山田太郎」がログイン済み
    And 会議室「渋谷ミーティングルームA」が「非公開」状態
    And キャンセルポリシーが未設定
    When 会議室公開管理画面で「公開可能」に変更を試みる
    Then 「キャンセルポリシーが未設定のため公開できません」とエラーが表示される

  Scenario: 他のオーナーの会議室の公開状態は変更できない
    Given 会議室オーナー「山田太郎」がログイン済み
    And 会議室「新宿会議室B」はオーナー「佐藤花子」の所有
    When 会議室「新宿会議室B」の公開管理画面にアクセスする
    Then 「この会議室の操作権限がありません」とエラーが表示される
```
