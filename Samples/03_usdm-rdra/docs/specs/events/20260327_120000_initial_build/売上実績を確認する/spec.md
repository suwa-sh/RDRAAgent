# 売上実績を確認する

## 概要

会議室オーナーが自身の会議室の売上実績を確認する。会議室ごとの利用回数・売上金額の実績データを参照し、運営改善に活用する。

## 関連 RDRA モデル

| モデル種別 | 要素名 | 関連 |
|-----------|--------|------|
| BUC | 利用実績管理フロー | このUCを含むBUC |
| アクター | 会議室オーナー | 操作するアクター |
| 情報 | 売上実績 | 参照する情報（会議室ごとの利用回数・売上金額） |

## E2E 完了条件（BDD）

### 正常系

```gherkin
Feature: 売上実績を確認する

  Scenario: 自身の会議室の売上実績を確認する
    Given 会議室オーナー「鈴木一郎」がログイン済み
    And 会議室「新宿カンファレンスルームA」の2026年3月の売上金額が150,000円・利用回数が10回である
    When 売上実績確認画面にアクセスする
    Then 会議室「新宿カンファレンスルームA」の売上金額150,000円・利用回数10回が表示される

  Scenario: 複数の会議室の売上実績を確認する
    Given 会議室オーナー「鈴木一郎」がログイン済み
    And 会議室「新宿カンファレンスルームA」の売上金額が150,000円である
    And 会議室「渋谷ミーティングルームB」の売上金額が80,000円である
    When 売上実績確認画面にアクセスする
    Then 全会議室の売上実績が一覧で表示される
    And 売上金額合計が230,000円と表示される

  Scenario: 期間を指定して売上実績を確認する
    Given 会議室オーナー「鈴木一郎」がログイン済み
    When 対象期間を「2026年1月」から「2026年3月」に指定して売上実績確認画面を表示する
    Then 指定期間内の会議室ごとの売上実績が表示される
```

### 異常系

```gherkin
  Scenario: 売上実績がない場合
    Given 会議室オーナー「鈴木一郎」がログイン済み
    And 売上実績が存在しない
    When 売上実績確認画面にアクセスする
    Then 「売上実績がありません」とメッセージが表示される

  Scenario: 他のオーナーの売上実績にはアクセスできない
    Given 会議室オーナー「鈴木一郎」がログイン済み
    When 他のオーナー「田中太郎」の会議室の売上実績にアクセスしようとする
    Then 「アクセス権限がありません」とエラーが表示される
```
