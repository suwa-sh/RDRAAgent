# オーナーを評価する

## 概要

利用者が会議室オーナーの対応などを評価登録する。オーナー評価情報が作成され、他の利用者の意思決定支援や会議室品質の可視化に活用される。

## 関連 RDRA モデル

| モデル種別 | 要素名 | 関連 |
|-----------|--------|------|
| BUC | 会議室評価フロー | このUCを含むBUC |
| アクター | 利用者 | 操作するアクター |
| 情報 | 会議室評価 | 作成する評価情報（オーナー評価として） |
| バリエーション | 評価種別 | オーナー評価 |

## E2E 完了条件（BDD）

### 正常系

```gherkin
Feature: オーナーを評価する

  Scenario: 会議室オーナーを評価する
    Given 利用者「田中太郎」がログイン済み
    And 会議室「新宿カンファレンスルームA」のオーナー「佐藤花子」の会議室利用が完了している
    When 評価スコア「5」、コメント「丁寧な対応でとても良かったです」でオーナー評価を登録する
    Then オーナー評価が作成される
    And 評価種別が「オーナー評価」である
    And 評価登録完了メッセージが表示される

  Scenario: 評価スコアのみでオーナーを評価する（コメントなし）
    Given 利用者「田中太郎」がログイン済み
    And 会議室利用が完了している
    When 評価スコア「3」のみでオーナー評価を登録する
    Then オーナー評価が作成される
```

### 異常系

```gherkin
  Scenario: 利用していない会議室のオーナーを評価しようとする
    Given 利用者「田中太郎」がログイン済み
    And 会議室「渋谷ミーティングスペースB」を利用したことがない
    When オーナーを評価しようとする
    Then 「利用済みの会議室のオーナーのみ評価できます」とエラーが表示される

  Scenario: 評価スコアが範囲外の場合
    Given 利用者「田中太郎」がログイン済み
    And 会議室利用が完了している
    When 評価スコア「0」でオーナー評価を登録しようとする
    Then 「評価スコアは1〜5の範囲で入力してください」とエラーが表示される
```
