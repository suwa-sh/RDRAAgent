# 問合せに対応する

## 概要

サービス運営担当者が利用者からのサービス問合せに対応し回答する。問合せの状態を「回答済み」から「対応済み」に遷移させ、問合せ対応を完了する。

## 関連 RDRA モデル

| モデル種別 | 要素名 | 関連 |
|-----------|--------|------|
| BUC | 問合せ管理フロー | このUCを含むBUC |
| アクター | サービス運営担当者 | 操作するアクター |
| 情報 | 問合せ | 参照・更新する情報（問合せへの対応・回答） |
| 状態 | 問合せ | 回答済みから対応済みへの状態遷移 |
| バリエーション | 問合せ種別 | サービス運営宛問合せ |

## E2E 完了条件（BDD）

### 正常系

```gherkin
Feature: 問合せに対応する

  Scenario: 回答済みの問合せを対応済みにする
    Given サービス運営担当者「山田花子」がログイン済み
    And 問合せ「Q001」の状態が「回答済み」である
    When 問合せ対応画面で問合せ「Q001」を「対応済み」に変更する
    Then 問合せ「Q001」の状態が「対応済み」に遷移する
    And 「問合せ対応を完了しました」と完了メッセージが表示される

  Scenario: 未対応の問合せ一覧を確認する
    Given サービス運営担当者「山田花子」がログイン済み
    And 未対応の問合せが3件、回答済みの問合せが2件存在する
    When 問合せ対応画面にアクセスする
    Then 問合せ一覧に全5件が表示される
    And 未対応の問合せが優先的に表示される

  Scenario: 問合せの詳細を確認する
    Given サービス運営担当者「山田花子」がログイン済み
    And 問合せ「Q001」が存在する
    When 問合せ「Q001」の詳細を表示する
    Then 問合せID・利用者名・問合せ内容・問合せ状態・問合せ日時・回答内容・回答日時が表示される
```

### 異常系

```gherkin
  Scenario: 未対応の問合せを対応済みにしようとした場合
    Given サービス運営担当者「山田花子」がログイン済み
    And 問合せ「Q002」の状態が「未対応」である
    When 問合せ対応画面で問合せ「Q002」を「対応済み」に変更しようとする
    Then 「回答済みの問合せのみ対応済みに変更できます」とエラーが表示される

  Scenario: サービス運営担当者以外がアクセスした場合
    Given 利用者「佐藤次郎」がログイン済み
    When 問合せ対応画面にアクセスする
    Then 「アクセス権限がありません」とエラーが表示される
```
