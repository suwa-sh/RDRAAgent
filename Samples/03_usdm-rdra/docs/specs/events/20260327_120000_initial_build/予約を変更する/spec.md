# 予約を変更する

## 概要

利用者が確定済みの予約内容（利用日時等）を変更する。予約状態が「確定」から「変更」に遷移し、オーナーの再許諾待ちとなる。

## 関連 RDRA モデル

| モデル種別 | 要素名 | 関連 |
|-----------|--------|------|
| BUC | 会議室予約フロー | このUCを含むBUC |
| アクター | 利用者 | 操作するアクター |
| 情報 | 予約情報 | 変更する予約情報 |
| 状態 | 予約 | 予約状態を「確定」から「変更」に遷移 |

## E2E 完了条件（BDD）

### 正常系

```gherkin
Feature: 予約を変更する

  Scenario: 確定済みの予約の利用日時を変更する
    Given 利用者「田中太郎」がログイン済み
    And 予約「R001」が「確定」状態で存在する（利用日時: 2026-04-10 10:00〜12:00）
    When 予約「R001」の利用日時を「2026-04-11 14:00〜16:00」に変更する
    Then 予約「R001」の状態が「変更」に遷移する
    And 変更後の利用日時が「2026-04-11 14:00〜16:00」に更新される
    And 予約変更完了メッセージが表示される
```

### 異常系

```gherkin
  Scenario: 申請状態の予約を変更しようとする
    Given 利用者「田中太郎」がログイン済み
    And 予約「R002」が「申請」状態で存在する
    When 予約「R002」の利用日時を変更しようとする
    Then 「確定済みの予約のみ変更できます」とエラーが表示される
    And 予約「R002」の状態は「申請」のままである

  Scenario: 変更先の日時が既に予約されている
    Given 利用者「田中太郎」がログイン済み
    And 予約「R001」が「確定」状態で存在する
    And 会議室に「2026-04-11 14:00〜16:00」の確定済み予約が別に存在する
    When 予約「R001」の利用日時を「2026-04-11 14:00〜16:00」に変更する
    Then 「指定日時は既に予約されています」とエラーが表示される
    And 予約「R001」の利用日時は変更されない

  Scenario: 過去の日時に変更しようとする
    Given 利用者「田中太郎」がログイン済み
    And 予約「R001」が「確定」状態で存在する
    When 予約「R001」の利用日時を「2025-01-01 10:00〜12:00」に変更する
    Then 「過去の日時は指定できません」とエラーが表示される
```
