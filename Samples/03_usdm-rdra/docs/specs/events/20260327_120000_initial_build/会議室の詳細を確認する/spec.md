# 会議室の詳細を確認する

## 概要

利用者が会議室の物件情報（広さ、収容人数、価格、設備・機能、画像）、運用ルール（利用可能時間帯、利用条件）、利用者からの評価を確認するユースケース。予約申請の前に会議室の詳細を把握するために利用される。

## 関連 RDRA モデル

| モデル種別 | 要素名 | 関連 |
|-----------|--------|------|
| BUC | 会議室予約フロー | このUCを含むBUC |
| アクター | 利用者 | 操作するアクター |
| 情報 | 会議室情報 | 参照する物件情報 |
| 情報 | 運用ルール | 参照する運用条件 |
| 情報 | 会議室評価 | 参照する評価情報 |

## E2E 完了条件（BDD）

### 正常系

```gherkin
Feature: 会議室の詳細を確認する

  Scenario: 利用者が会議室の物件情報を確認する
    Given 利用者「鈴木一郎」がログイン済み
    And 会議室「渋谷ミーティングルームA」が「公開中」状態
    When 会議室詳細画面を開く
    Then 会議室名「渋谷ミーティングルームA」が表示される
    And 所在地「東京都渋谷区渋谷1-1-1」が表示される
    And 広さ「30㎡」、収容人数「10名」、価格「3000円/時間」が表示される
    And 設備「プロジェクター, Wi-Fi, 空調」が表示される

  Scenario: 利用者が会議室の運用ルールを確認する
    Given 利用者「鈴木一郎」がログイン済み
    And 会議室「渋谷ミーティングルームA」に運用ルールが設定済み
    When 会議室詳細画面を開く
    Then 利用可能時間帯「9:00-21:00」が表示される
    And 最低利用時間「1時間」が表示される

  Scenario: 利用者が会議室の評価を確認する
    Given 利用者「鈴木一郎」がログイン済み
    And 会議室「渋谷ミーティングルームA」に平均評価スコア「4.2」の評価がある
    When 会議室詳細画面を開く
    Then 平均評価スコア「4.2」が表示される
    And 利用者からの評価コメントが表示される
```

### 異常系

```gherkin
  Scenario: 非公開の会議室は詳細を確認できない
    Given 利用者「鈴木一郎」がログイン済み
    And 会議室「渋谷ミーティングルームA」が「非公開」状態
    When 会議室詳細画面にアクセスする
    Then 「この会議室は現在公開されていません」とエラーが表示される

  Scenario: 存在しない会議室の詳細を確認しようとした場合
    Given 利用者「鈴木一郎」がログイン済み
    When 存在しない会議室IDで詳細画面にアクセスする
    Then 「会議室が見つかりません」とエラーが表示される
```
