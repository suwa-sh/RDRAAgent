| ID | 要求 | 仕様 | 理由（背景） | 説明 |
|----|------|------|------------|------|
| REQ-001 | 会議室オーナーの登録・管理機能を提供する | | 会議室を貸し出すオーナーがサービスに参加し、プロフィールや規約確認を通じて信頼性を担保するため | |
| SPEC-001-01 | | オーナー登録機能（規約参照・確認、プロフィール登録を含む）を提供する | | 1. Given 未登録ユーザー When オーナー登録画面で必要情報を入力し規約に同意する Then オーナー申請が完了し審査待ち状態になる<br>2. Given オーナー申請者 When プロフィール情報を入力する Then プロフィールが保存される |
| SPEC-001-02 | | オーナー退会機能を提供する | | 1. Given 登録済みオーナー When 退会申請を行う Then オーナーアカウントが退会状態になり貸出中の会議室が非公開になる |
| REQ-002 | 会議室の登録・管理機能を提供する | | オーナーが所有する会議室の物件情報や運用ルールを登録し、利用者に貸し出せる状態にするため | |
| SPEC-002-01 | | 会議室登録機能（物件情報：広さ・価格・機能性、運用ルール：キャンセルポリシー・貸出可否）を提供する | | 1. Given 登録済みオーナー When 会議室情報（広さ、価格、機能性）と運用ルール（キャンセルポリシー、貸出可否）を入力する Then 会議室が登録され公開可能状態になる |
| SPEC-002-02 | | 会議室評価の確認機能を提供する | | 1. Given 登録済みオーナー When 自身の会議室一覧を表示する Then 各会議室の利用者評価が確認できる |
| REQ-003 | 会議室の貸出機能を提供する | | オーナーが利用者の申請を審査し、鍵の貸出・返却を通じて会議室利用を管理するため | |
| SPEC-003-01 | | 利用者の使用許諾機能（過去の利用評価確認を含む）を提供する | | 1. Given 会議室予約申請がある When オーナーが利用者の過去評価を確認し許諾する Then 予約が確定状態になる |
| SPEC-003-02 | | 会議室の鍵貸出・返却管理機能を提供する | | 1. Given 予約確定状態 When オーナーが利用者に鍵を貸し出す Then 会議室利用が開始状態になる<br>2. Given 会議室利用中 When 利用者が鍵を返却する Then 会議室利用が終了状態になる |
| SPEC-003-03 | | 利用者評価登録機能と利用者からの問合せ回答機能を提供する | | 1. Given 会議室利用終了後 When オーナーが利用者の評価を登録する Then 利用者評価が記録される<br>2. Given 利用者から問合せがある When オーナーが回答する Then 問合せが回答済みになる |
| REQ-004 | 会議室の利用実績・売上管理機能を提供する | | オーナーが自身の会議室の利用状況と売上を把握し、運営改善に活用するため | |
| SPEC-004-01 | | 会議室評価の確認と売上実績管理機能を提供する | | 1. Given 登録済みオーナー When 利用実績画面を表示する Then 会議室別の利用回数・売上実績・利用者評価が確認できる |
| REQ-005 | サービス運営管理機能を提供する | | サービス運営担当者が手数料売上分析、オーナー審査、運用管理を行いサービス品質を維持するため | |
| SPEC-005-01 | | 手数料売上分析機能（会議室別・貸出別）を提供する | | 1. Given サービス運営担当者 When 売上分析画面を表示する Then 会議室別・貸出別の手数料売上が確認できる |
| SPEC-005-02 | | オーナー登録審査機能を提供する | | 1. Given オーナー申請がある When サービス運営担当者が審査し承認する Then オーナーが登録済み状態になる<br>2. Given オーナー申請がある When サービス運営担当者が審査し却下する Then オーナーが却下状態になる |
| SPEC-005-03 | | 利用状況管理機能（利用履歴管理：会員別・物件別、会議室利用状況分析）を提供する | | 1. Given サービス運営担当者 When 利用状況画面を表示する Then 会員別・物件別の利用履歴と分析結果が確認できる |
| SPEC-005-04 | | 利用者からのサービス問合せ対応機能を提供する | | 1. Given 利用者からサービスへの問合せがある When サービス運営担当者が対応する Then 問合せが対応済みになる |
| REQ-006 | 会議室オーナーへの精算機能を提供する | | 会議室利用料からサービス手数料を差し引いた金額をオーナーに支払い、収益分配を実現するため | |
| SPEC-006-01 | | 会議室別の利用履歴からオーナーへの精算額を計算し支払う機能を提供する | | 1. Given 月末 When 精算処理を実行する Then 会議室別利用履歴に基づきオーナーへの精算額が計算され決済機関を通じて支払われる |
| REQ-007 | 利用者の会議室予約機能を提供する | | 利用者が条件に合う会議室を検索・予約し、決済手段を設定して利用できるようにするため | |
| SPEC-007-01 | | 会議室照会機能（広さ・価格・機能性・評価での検索）を提供する | | 1. Given 利用者 When 会議室検索条件（広さ、価格、機能性、評価）を入力する Then 条件に合致する会議室一覧が表示される |
| SPEC-007-02 | | 会議室予約機能（決済方法設定：クレジットカード・電子マネー）を提供する | | 1. Given 利用者が会議室を選択 When 予約日時と決済方法（クレジットカードまたは電子マネー）を設定し予約する Then 予約が申請状態になる |
| SPEC-007-03 | | 予約の変更・取消機能を提供する | | 1. Given 予約済み利用者 When 予約内容を変更する Then 予約情報が更新される<br>2. Given 予約済み利用者 When 予約を取り消す Then キャンセルポリシーに基づき予約が取消状態になる |
| SPEC-007-04 | | 会議室オーナーへの問合せ機能を提供する | | 1. Given 利用者 When 会議室に関する問合せを送信する Then オーナーに問合せが通知される |
| REQ-008 | 利用した会議室の評価機能を提供する | | 利用者が会議室とオーナーを評価し、他の利用者の意思決定を支援するため | |
| SPEC-008-01 | | 会議室とオーナーへの評価登録機能を提供する | | 1. Given 会議室利用終了後の利用者 When 会議室とオーナーの評価を登録する Then 評価が保存され会議室情報に反映される |
| REQ-009 | 支払と精算のサービスポリシーを適用する | | 予約時の決済登録、利用後の引き落とし、月末のオーナー精算という支払フローを統一的に管理するため | |
| SPEC-009-01 | | 予約時に決済手段を登録し、利用後にサービス運営者が引き落とし、月末にオーナーへ支払う支払フローを実装する | | 1. Given 利用者が予約する When 決済手段を登録する Then 決済情報が予約に紐付けられる<br>2. Given 会議室利用終了後 When サービス運営者が利用料を引き落とす Then 利用者の決済手段から利用料が徴収される<br>3. Given 月末 When オーナーへの精算を実行する Then 手数料を差し引いた利用料がオーナーに支払われる |
| REQ-010 | 会議室利用のサービスポリシーを適用する | | 鍵の受け渡しをトリガーとした会議室利用の開始・終了を明確に定義し、利用状態を管理するため | |
| SPEC-010-01 | | 利用者が会議室に行き鍵を受け取ることで利用開始、鍵を返却することで利用終了とするポリシーを適用する | | 1. Given 予約確定状態の利用者 When 会議室オーナーから鍵を受け取る Then 会議室利用が開始状態になる<br>2. Given 会議室利用中 When 鍵を返却する Then 会議室利用が終了状態になる |
| REQ-011 | 会議室オーナーがバーチャル会議室を登録・管理できるようにする | | コロナ禍以降オンライン会議のニーズが増加しており、物理会議室だけでなくZoomやTeamsなどのオンライン会議室も貸し出し対象として扱えるようにするため | |
| SPEC-011-01 | | バーチャル会議室の登録機能を提供する。会議ツール種別、同時接続数、録画可否などバーチャル会議室固有の属性を登録できるようにする | | 1. Given 登録済みオーナー When バーチャル会議室登録画面で会議ツール種別・同時接続数・録画可否・価格を入力して登録する Then バーチャル会議室が登録され公開可能状態になる<br>2. Given 登録済みオーナー When 会議ツール種別としてZoom・Teams・Google Meetなどを選択できる Then 選択した会議ツール種別がバーチャル会議室情報に保存される |
| SPEC-011-02 | | 会議室情報に会議室種別（物理・バーチャル）を追加し、種別に応じて登録項目を切り替えられるようにする | | 1. Given オーナーが会議室を登録する When 会議室種別として「バーチャル」を選択する Then 所在地・広さなどの物理属性の代わりに会議ツール種別・同時接続数・録画可否が入力項目として表示される<br>2. Given オーナーが会議室を登録する When 会議室種別として「物理」を選択する Then 従来通り所在地・広さ・収容人数などが入力項目として表示される |
| REQ-012 | 利用者がバーチャル会議室を検索・予約できるようにする | | 利用者がオンライン会議のニーズに合ったバーチャル会議室を条件指定で探し出し、予約できるようにするため | |
| SPEC-012-01 | | 会議室検索条件に会議室種別（物理・バーチャル）、会議ツール種別、同時接続数を追加し、バーチャル会議室を検索できるようにする | | 1. Given 利用者 When 会議室検索画面で会議室種別「バーチャル」を指定して検索する Then バーチャル会議室のみが検索結果に表示される<br>2. Given 利用者 When 会議ツール種別「Zoom」と同時接続数「50人以上」を指定して検索する Then 条件に合致するバーチャル会議室一覧が表示される |
| SPEC-012-02 | | バーチャル会議室の予約機能を提供する。決済フローは物理会議室と同じフローを使用する | | 1. Given 利用者がバーチャル会議室を選択 When 予約日時と決済方法を設定し予約する Then 予約が申請状態になり、物理会議室と同じ決済フローで処理される |
| REQ-013 | バーチャル会議室の予約確定時に会議URLを利用者に自動通知する | | バーチャル会議室では物理的な鍵の受け渡しが不要であり、代わりに会議URLの自動通知によってスムーズに利用を開始できるようにするため | |
| SPEC-013-01 | | バーチャル会議室の予約確定時に、オーナーが設定した会議URLを利用者に自動通知する機能を提供する | | 1. Given バーチャル会議室の予約がオーナーに許諾された When 予約が確定状態に遷移する Then 会議URLが利用者にメール等で自動通知される<br>2. Given 物理会議室の予約が確定した When 予約が確定状態に遷移する Then 会議URL通知は行われない（従来通り鍵の受け渡しフロー） |
| SPEC-013-02 | | バーチャル会議室では鍵の貸出・返却を不要とし、予約確定で利用開始、利用終了時刻到来で利用終了とする利用フローを提供する | | 1. Given バーチャル会議室の予約が確定 When 利用開始時刻になる Then 会議室利用が自動的に利用開始状態になる（鍵の貸出記録は不要）<br>2. Given バーチャル会議室が利用中 When 利用終了時刻になる Then 会議室利用が自動的に利用終了状態になる（鍵の返却記録は不要） |
| REQ-014 | バーチャル会議室にも評価機能を適用する | | 利用者がバーチャル会議室の品質（接続安定性、音質、操作性など）を評価し、他の利用者の意思決定を支援するため | |
| SPEC-014-01 | | バーチャル会議室の利用終了後に、利用者がバーチャル会議室とオーナーを評価できるようにする。既存の会議室評価フローを適用する | | 1. Given バーチャル会議室の利用終了後の利用者 When 会議室評価を登録する Then バーチャル会議室の評価が保存され会議室情報に反映される<br>2. Given バーチャル会議室の利用終了後の利用者 When オーナー評価を登録する Then オーナーの評価が保存される |
