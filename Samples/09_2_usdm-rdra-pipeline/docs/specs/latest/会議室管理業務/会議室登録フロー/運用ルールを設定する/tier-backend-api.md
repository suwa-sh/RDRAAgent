# 運用ルールを設定する - バックエンドAPI仕様

## 変更概要

運用ルールを設定するのAPIエンドポイントを実装する。運用ルールを登録する。

## API 仕様

### 運用ルール設定

- **メソッド**: PUT
- **パス**: /api/v1/rooms/:id/rules
- **認証**: Bearer JWT
- **OpenAPI**: [openapi.yaml](../../../_cross-cutting/api/openapi.yaml) の paths./api/v1/rooms/{id}/rules.put を参照

#### リクエスト

| パラメータ | 型 | 必須 | 説明 |
|-----------|---|------|------|
| (リクエストボディ) | UpdateRulesRequest | Yes | リクエストデータ |

#### レスポンス

| フィールド | 型 | 説明 |
|-----------|---|------|
| data | RulesResponse | レスポンスデータ |

#### エラーレスポンス

| ステータスコード | 条件 | レスポンス |
|----------------|------|-----------|
| 400 | バリデーションエラー | {code: "VALIDATION_ERROR", message: "入力値が不正です"} |
| 401 | 認証エラー | {code: "UNAUTHORIZED", message: "認証が必要です"} |
| 403 | 認可エラー | {code: "FORBIDDEN", message: "権限がありません"} |
| 404 | リソース未存在 | {code: "NOT_FOUND", message: "リソースが見つかりません"} |

## データモデル変更

### operation_rules

| カラム | 型 | 説明 | 変更種別 |
|--------|---|------|---------|
| (運用ルールの属性) | 情報.tsvの定義 | 運用ルールのデータ | 追加 |

### rooms

| カラム | 型 | 説明 | 変更種別 |
|--------|---|------|---------|
| (会議室情報の属性) | 情報.tsvの定義 | 会議室情報のデータ | 更新 |

## ビジネスルール

- キャンセルポリシー: 条件.tsvの定義に従う
- 状態遷移: 会議室状態 公開中 → 貸出停止（貸出停止設定）
- 状態遷移: 会議室状態 貸出停止 → 公開中（貸出再開設定）
