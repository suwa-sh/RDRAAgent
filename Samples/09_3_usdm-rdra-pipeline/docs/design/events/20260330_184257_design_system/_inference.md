- 売上実績
- 精算情報

## 4. 状態モデル (Badge/Tracker)
- オーナー申請状態: 未申請→申請中→審査中→承認/却下→退会 (6状態 → StepTracker)
- 予約状態: 仮予約→確定→利用中→利用完了 / 変更 / 取消 (7状態 → StepTracker)
- 問合せ状態: 受付→対応中→回答済→完了 (4状態 → StepTracker)

## 5. バリエーション → フィルター/フォーム
- 決済方法: クレジットカード/電子マネー → ボタングループ
- 貸出可否: 貸出可/貸出不可 → ボタントグル
- 問合せ種別: 会議室問合せ/サービス問合せ → ボタングループ
- 評価種別: 会議室評価/ホスト評価/利用者評価 → タブ

## 6. NFR → デザイン判断
- 可用性 (24h運用 Lv3): エラー状態・ローディング状態を全画面に
- 性能 (同時1万 Lv3): ページネーション必須
- 性能 (5秒以内 Lv3): Skeleton UI でパーシーブドパフォーマンス改善
- セキュリティ (RBAC Lv2): ポータル別ナビゲーション
- 運用 (モバイル対応): User ポータルをモバイルファースト設計

## 7. Arch → 技術判断
- フレームワーク: Next.js + TypeScript → React コンポーネント
- ポータル3つ: data-portal CSS セレクタで切替
- SSR 対応: 検索画面等は Skeleton UI を考慮
- レスポンシブ: モバイル・デスクトップ両対応
