## 全体横断設計方針

### ユーザーフロー
1. 利用者フロー: 会議室検索 -> 予約 -> 鍵受取 -> 利用 -> 評価
2. オーナーフロー: 規約確認 -> 登録 -> 申請 -> 会議室登録 -> 貸出管理 -> 精算確認
3. 運営フロー: オーナー審査 -> 運用管理 -> 利用分析 -> 精算処理

### 情報アーキテクチャ
- user ポータル: 会議室検索, 予約管理, 鍵管理, 評価, 問合せ
- owner ポータル: 規約, オーナー情報, 会議室管理, 予約/鍵管理, 評価確認, 精算, 問合せ
- admin ポータル: オーナー審査, 運用管理, 利用分析, 精算管理, 問合せ対応

### データ可視化対象
- 手数料分析画面: 手数料売上の時系列推移 (Trend)
- 運用管理ダッシュボード: 各種KPI (Comparison)
- 利用状況分析画面: 会議室別利用率 (Composition)
