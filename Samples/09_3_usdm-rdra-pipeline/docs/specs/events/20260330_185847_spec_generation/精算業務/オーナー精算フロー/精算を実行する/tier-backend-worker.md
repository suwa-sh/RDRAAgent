# 精算を実行する - バックエンドワーカー仕様

## 変更概要

算出した精算額を決済機関経由でオーナーに支払うのバッチ処理/非同期処理を実装する。

## イベント処理仕様

### 精算を実行するWorker

- **トリガー**: 精算額計算完了イベント
- **入力チャネル**: settlement-calculation-completed
- **出力チャネル**: settlement-payment-notifications

#### 処理フロー

1. 計算済み精算情報を取得する
2. 決済機関APIに精算支払リクエストを送信する
3. 精算結果を記録する
4. 処理完了ログを出力する

#### エラーハンドリング

| エラー種別 | リトライ | DLQ | 説明 |
|-----------|---------|-----|------|
| DB接続エラー | Yes(3回) | No | 一時的な接続障害。指数バックオフでリトライ |
| 決済機関APIエラー | Yes(3回) | Yes | Circuit Breaker発動後にDLQに退避 |
| タイムアウト | Yes(1回) | Yes | 処理時間超過。再実行後もタイムアウトならDLQへ |

## データモデル変更

### settlements

| カラム | 型 | 説明 | 変更種別 |
|--------|---|------|---------|
| settlement_id | uuid | 精算ID | 既存 |
| status | string | 精算状態(pending/calculated/executed/failed) | 更新 |
| calculated_at | datetime | 計算日時 | 更新 |

## ビジネスルール

- 冪等性確保: ジョブ実行IDで重複実行を検知
- バッチ処理時間制限: 8時間以内に完了すること (NFR B.2.2.1)
- 個別エラーがバッチ全体を停止させない
