## チャート選定ガイドライン

### 観点別チャート選定

| 観点 | 推奨チャート | 使用場面 |
|------|-----------|---------|
| Comparison（比較） | 棒グラフ（横） | 会議室別売上、物件別稼働率、オーナー別精算額 |
| Composition（構成比） | 円グラフ | 会議室カテゴリ別売上構成比（5項目以下の場合） |
| Trend（傾向） | 折れ線グラフ | 月次売上推移、利用件数推移、精算額推移 |
| Distribution（分布） | ヒストグラム | 会員別利用頻度分布 |

### データ表現フレームワーク

| 表現方法 | 適用画面 | 具体例 |
|---------|---------|--------|
| % Change（変化率） | 売上分析画面 | 前月比 +15.3% の SummaryCard |
| Variance（差異） | 利用状況分析画面 | 目標稼働率との差分表示 |
| Calculated Metric（計算指標） | 売上分析画面 | 1会議室あたり平均売上 |
| Added Context（追加コンテキスト） | 売上分析画面 | 業界平均ラインの折れ線追加 |

## ダッシュボード設計原則

### 情報の階層化

- **全体サマリー**: SummaryCard 4枚で KPI 一覧（売上合計、利用件数、稼働率、精算待ち件数）
- **ドリルダウン**: SummaryCard クリックで詳細チャートに遷移。テーブルの行クリックで個別詳細に遷移
- **フィルター**: 期間選択（今月/先月/3ヶ月/6ヶ月/1年/カスタム）、会議室フィルター、オーナーフィルター

### データストーリーテリング

- **ナラティブ**: 「今月の売上は前月比{+N}%で{増加/減少}傾向」の自動コメント
- **比較軸**: 前月比、前年同月比
- **アクション**: 売上低下時は「稼働率の低い会議室を確認」のリンク表示

## 認知負荷への配慮

- ダッシュボード上の KPI は 4 項目まで（SummaryCard 4枚）
- チャートは画面あたり最大 3 個（スクロール不要）
- Data-Ink Ratio: グリッド線は最小限、背景は var(--semantic-background) のフラット、3D 効果なし
- ゲシュタルトの近接法則: 関連する KPI カードを横並び配置、チャートは縦並びで区分
- チャートのカラーパレットは design-event.yaml のトークンから選定:
  - プライマリ系列: var(--color-blue-600)
  - セカンダリ系列: var(--color-teal-600)
  - 警告系列: var(--color-orange-500)
  - 成功系列: var(--color-green-600)
