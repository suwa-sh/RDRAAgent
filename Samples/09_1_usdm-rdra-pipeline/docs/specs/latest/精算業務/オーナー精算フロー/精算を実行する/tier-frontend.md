# 精算を実行する - フロントエンド仕様

## 変更概要

Admin ポータルに精算実行画面を実装する。精算額確認と実行確認ダイアログを提供する。

## 画面仕様

### 精算実行画面

- **URL**: /admin/settlements/:id/execute
- **アクセス権**: サービス運営担当者
- **ポータル**: admin

#### 表示要素とコンポーネントマッピング

| 要素 | 種別 | デザインシステムコンポーネント | 説明 |
|------|------|------------------------------|------|
| 精算額 | テキスト | PriceDisplay | 精算額のJPY表示 |
| オーナー情報 | カード | Card | オーナー名、対象月 |
| 実行ボタン | ボタン | Button (default) | 精算実行アクション |
| 確認ダイアログ | モーダル | ConfirmActionModal | 精算実行前の確認 |

#### デザイントークン参照

| 用途 | トークン | 値 |
|------|---------|---|
| 背景色 | var(--semantic-background) | #FFFFFF |
| アクセント | var(--color-slate-700) | #334155 |
| 警告色 | var(--semantic-warning) | var(--color-orange-500) |

#### UIロジック

- **状態管理**: settlement, isExecuting, executionResult
- **バリデーション**: 精算額が計算済みであること
- **ローディング**: 実行中はボタンdisabled + スピナー（ダブルクリック防止）
- **エラーハンドリング**: 実行エラー時はリトライボタン表示

#### 操作フロー

1. 精算処理画面から遷移、精算情報を表示
2. PriceDisplay で精算額を確認
3. 「精算を実行する」ボタンクリック
4. ConfirmActionModal で確認（「オーナー「{名前}」に{金額}円を精算します。実行しますか？」）
5. 確認後にAPI呼出し（冪等キー付き）
6. 「処理中」状態を表示
