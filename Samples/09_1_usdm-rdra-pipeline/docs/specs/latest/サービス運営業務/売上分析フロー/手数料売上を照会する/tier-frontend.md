# 手数料売上を照会する - フロントエンド仕様

## 変更概要

adminポータルに売上分析画面を実装する。

## 画面仕様

### 売上分析画面

- **URL**: /admin/analytics/revenue
- **アクセス権**: サービス運営担当者
- **ポータル**: admin

#### 表示要素とコンポーネントマッピング

| 要素 | 種別 | デザインシステムコンポーネント | 説明 |
|------|------|------------------------------|------|
| 主要コンテンツ | フォーム/表示 | Card + Input/Button | 手数料売上を照会するの主要UI |
| アクションボタン | ボタン | Button (default) | 主要アクション |

#### デザイントークン参照

| 用途 | トークン | 値 |
|------|---------|---|
| 背景色 | var(--semantic-background) | #FFFFFF |
| アクセント | ポータルプライマリ | var(--color-slate-700) |

#### UIロジック

- **状態管理**: フォーム入力値、送信中フラグ、エラー状態
- **バリデーション**: 必須項目チェック
- **ローディング**: API呼出し中はSkeleton/スピナー表示
- **エラーハンドリング**: バリデーションエラーはインライン、APIエラーはトースト

#### 操作フロー

1. 画面表示時にデータ取得（該当する場合）
2. ユーザーが情報を入力/選択
3. アクションボタンで処理実行
4. 完了後にフィードバック表示
