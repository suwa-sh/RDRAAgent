## 4. NFR/Arch からの設計判断

| NFR/Arch 要素 | 設計判断 |
|--------------|---------|
| 可用性 (24h近い運用) | エラー状態・ローディング状態を全画面に実装 |
| 性能 (同時100ユーザー) | ページネーション実装、仮想スクロール不要 |
| 性能 (3秒以内レスポンス) | Skeleton UI でパーシーブドパフォーマンス改善 |
| セキュリティ (RBAC) | ポータル別ナビゲーション、権限ベース表示切替 |
| セキュリティ (MFA) | MFA 設定画面を考慮 |
| presentation: Next.js | React + TypeScript コンポーネント |
| ポータル3種 | data-portal 属性でテーマ切替 |
| IdP (マネージド) | ログイン画面はカスタム UI |
| モバイル対応 (NFR F.1.1.3) | User ポータルをモバイルファースト設計 |
