# 精算額を計算する - バックエンドワーカー仕様

## 変更概要

会議室別の利用履歴から精算額を算出するのバッチ処理/非同期処理を実装する。

## イベント処理仕様

### 精算額を計算するWorker

- **トリガー**: CronJob（月末実行）
- **入力チャネル**: スケジュール起動
- **出力チャネル**: settlement-calculation-completed

#### 処理フロー

1. 対象期間の全オーナーを取得する
2. オーナーごとに利用履歴を集計し手数料を控除して精算額を算出する
3. 精算情報をDBに保存する
4. 処理完了ログを出力する

#### エラーハンドリング

| エラー種別 | リトライ | DLQ | 説明 |
|-----------|---------|-----|------|
| DB接続エラー | Yes(3回) | No | 一時的な接続障害。指数バックオフでリトライ |
| 計算エラー | Yes(3回) | Yes | 不正データをDLQに退避して後続処理を継続 |
| タイムアウト | Yes(1回) | Yes | 処理時間超過。再実行後もタイムアウトならDLQへ |

## データモデル変更

### settlements

| カラム | 型 | 説明 | 変更種別 |
|--------|---|------|---------|
| settlement_id | uuid | 精算ID | 既存 |
| status | string | 精算状態(pending/calculated/executed/failed) | 更新 |
| calculated_at | datetime | 計算日時 | 更新 |

## ビジネスルール

- 精算条件: RDRA条件定義に基づく精算額計算ロジック
- バッチ処理時間制限: 8時間以内に完了すること (NFR B.2.2.1)
- 個別エラーがバッチ全体を停止させない

## ティア完了条件（BDD）

