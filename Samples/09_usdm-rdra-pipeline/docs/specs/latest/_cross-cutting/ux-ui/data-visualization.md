## チャート選定ガイドライン

### 観点別チャート選定

| 観点 | 推奨チャート | 使用場面 |
|------|-----------|---------|
| Comparison（比較） | 棒グラフ（横棒/縦棒） | 会議室別売上比較、オーナー別精算額比較 |
| Composition（構成比） | ドーナツチャート、積み上げ棒グラフ | 手数料率別構成、問合せ種別構成 |
| Relationship（関連性） | 散布図 | 価格 vs 利用率の相関（将来拡張） |
| Distribution（分布） | ヒートマップ | 時間帯別利用分布 |
| Trend（傾向） | 折れ線グラフ、エリアチャート | 月別予約推移、売上推移 |

## ダッシュボード設計原則

### 情報の階層化

- **全体サマリー**: KPIカード4-5枚をページ上部に配置（予約数、利用中数、オーナー数、売上合計、問合せ件数）
- **ドリルダウン**: KPIカードクリックで詳細画面に遷移。チャートのデータポイントクリックでフィルター適用
- **フィルター**: 期間選択（今日/今週/今月/カスタム）をページ上部に常時表示

### データストーリーテリング

- **ナラティブ**: 前期比の増減を矢印アイコン + パーセンテージで表示
- **比較軸**: 前月比をデフォルト。前年比は切替可能
- **アクション**: 異常値（前月比50%以上増減）にアラートバッジを表示し、詳細確認を促す

## 認知負荷への配慮

- ワーキングメモリの限界（4-5項目）を考慮し、ダッシュボードの KPI カード数は5枚以下
- Data-Ink Ratio を意識し、グリッド線は最小限。凡例はチャート内に直接ラベル配置
- ゲシュタルトの法則に基づき、関連チャートをカードでグルーピング
- 数値は3桁区切り + 単位（件、円、%）を必ず表示
- 大きな数値は省略表記（10,000 -> 1万、1,000,000 -> 100万）
