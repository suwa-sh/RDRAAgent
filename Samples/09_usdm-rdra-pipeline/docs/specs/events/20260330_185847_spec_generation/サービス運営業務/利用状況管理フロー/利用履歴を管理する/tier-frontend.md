# 利用履歴を管理する - フロントエンド仕様

## 変更概要

利用履歴管理画面を実装し、サービス運営担当者が会員別・物件別の利用履歴を管理するための画面を提供する。

## 画面仕様

### 利用履歴管理画面

- **URL**: /admin/usage/history
- **アクセス権**: サービス運営担当者
- **ポータル**: admin

#### 表示要素とコンポーネントマッピング

| 要素 | 種別 | デザインシステムコンポーネント | 説明 |
|------|------|------------------------------|------|
| Card | コンポーネント | Card | design-event.yaml定義済み |
| Badge | コンポーネント | Badge | design-event.yaml定義済み |

#### デザイントークン参照

| 用途 | トークン | 値 |
|------|---------|---|
| 背景色 | var(--semantic-background) | #FFFFFF |
| アクセント | var(--portal-primary) | #475569 |
| ボーダー | var(--semantic-border) | var(--color-gray-200) |

#### UIロジック

- **状態管理**: React Server Components + クライアント状態管理
- **バリデーション**: フロントエンドバリデーション（必須チェック、形式チェック）
- **ローディング**: Skeleton UIでパーシーブドパフォーマンス改善
- **エラーハンドリング**: ErrorBannerコンポーネントでエラーメッセージ表示、リトライボタン付き

#### 操作フロー

1. サービス運営担当者が利用履歴管理画面にアクセスする
2. Cardが表示される
3. 会員別・物件別の利用履歴を管理する
4. 操作完了のフィードバックが表示される

## コンポーネント設計

### Card

- **ベースコンポーネント**: Card
- **Props**:
  | Prop | 型 | 必須 | 説明 |
  |------|---|------|------|
  | data | object | Yes | 表示データ |
- **状態**: ローディング、エラー、成功
- **イベント**: onChange, onSubmit

### Badge

- **ベースコンポーネント**: Badge
- **Props**:
  | Prop | 型 | 必須 | 説明 |
  |------|---|------|------|
  | data | object | Yes | 表示データ |
- **状態**: ローディング、エラー、成功
- **イベント**: onChange, onSubmit
