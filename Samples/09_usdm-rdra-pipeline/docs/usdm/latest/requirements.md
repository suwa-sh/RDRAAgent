| SPEC-001-02 | | オーナーが規約を参照・確認できる | | 1. Given オーナー登録画面 When 規約参照リンクを押す Then サービス規約が表示され確認できる |

| SPEC-001-04 | | オーナーが退会できる | | 1. Given 承認済みのオーナー When 退会手続きを行う Then オーナーアカウントが退会状態になる |
| SPEC-001-05 | | オーナーがプロフィールを登録・編集できる | | 1. Given 登録済みのオーナー When プロフィール編集画面で情報を更新する Then プロフィール情報が反映される |
| REQ-002 | 会議室オーナーが会議室を登録し運用ルールを設定できること | | 貸し会議室サービスの中核機能として、オーナーが物件情報を登録し貸出条件を管理できる必要がある | |
| SPEC-002-01 | | 会議室の物件情報（広さ、価格、機能性）を登録できる | | 1. Given 承認済みのオーナー When 会議室登録画面で物件情報を入力して登録する Then 会議室が一覧に表示される |
| SPEC-002-02 | | 運用ルール（キャンセルポリシー、貸出可否）を設定できる | | 1. Given 登録済みの会議室 When 運用ルール設定画面でキャンセルポリシーと貸出可否を設定する Then 運用ルールが会議室に紐づいて保存される |
| SPEC-002-03 | | 会議室の評価を確認できる | | 1. Given 貸出実績のある会議室 When オーナーが評価確認画面を開く Then 利用者からの評価一覧が表示される |
| REQ-003 | 会議室オーナーが会議室の貸出を管理できること | | 利用者への会議室貸出において、使用許諾・鍵管理・利用者評価・問合せ対応が円滑に行われる必要がある | |
| SPEC-003-01 | | 利用者の使用許諾を行う（過去の利用評価を確認した上で） | | 1. Given 予約申請があった利用者 When オーナーが利用者の過去評価を確認し許諾する Then 予約が確定状態になる |
| SPEC-003-02 | | 会議室の鍵を利用者に貸し出す | | 1. Given 許諾済みの予約 When 利用者が会議室に到着する Then オーナーが鍵を貸し出し、利用開始状態になる |
| SPEC-003-03 | | 利用者を評価してオーナー側の記録として登録する | | 1. Given 利用が終了した予約 When オーナーが利用者評価を入力する Then 利用者評価が記録される |
| SPEC-003-04 | | 利用者からの問合せに回答する | | 1. Given 利用者から問合せがあった場合 When オーナーが問合せ内容を確認して回答する Then 回答が利用者に届く |
| REQ-004 | 会議室オーナーが利用実績と売上を管理できること | | オーナーが自身の会議室の稼働状況と収益を把握し、運営改善に活かすため | |
| SPEC-004-01 | | 会議室ごとの利用者評価を確認できる | | 1. Given 利用実績のある会議室 When 利用実績画面を開く Then 利用者からの評価一覧が表示される |
| SPEC-004-02 | | 売上実績を管理できる | | 1. Given 貸出実績のある会議室 When 売上管理画面を開く Then 会議室別の売上実績が表示される |
| REQ-005 | サービス運営担当者がサービス全体を運営管理できること | | サービスの健全な運営のため、手数料管理・オーナー審査・利用状況分析・問合せ対応が必要である | |
| SPEC-005-01 | | 手数料売上を会議室別・貸出別に分析できる | | 1. Given サービス運営担当者がログイン済み When 手数料分析画面を開く Then 会議室別・貸出別の手数料売上が表示される |
| SPEC-005-02 | | オーナー登録を審査できる | | 1. Given オーナー申請がある場合 When 運営担当者が審査画面で承認または却下する Then オーナーの申請状態が更新される |
| SPEC-005-03 | | 利用履歴を会員別・物件別に管理できる | | 1. Given サービス運営担当者がログイン済み When 利用履歴管理画面を開く Then 会員別・物件別の利用履歴が表示される |
| SPEC-005-04 | | 会議室利用状況を分析できる | | 1. Given 利用実績データがある場合 When 利用状況分析画面を開く Then 会議室の利用状況の統計が表示される |
| SPEC-005-05 | | 利用者からのサービス問合せに対応できる | | 1. Given 利用者からサービスへの問合せがある場合 When 運営担当者が問合せ一覧で確認・対応する Then 問合せのステータスが更新される |
| REQ-006 | サービス運営担当者が会議室オーナーへの精算を行えること | | オーナーへの利用料支払いはサービス運営の根幹であり、会議室別の利用履歴から正確な精算額を算出する必要がある | |
| SPEC-006-01 | | 会議室別の利用履歴からオーナーへの精算額を計算し支払う | | 1. Given 月末 When 精算処理を実行する Then 会議室別の利用料から手数料を差し引いた金額がオーナーに支払われる |

| SPEC-007-01 | | 会議室を広さ・価格・機能性・評価で照会できる | | 1. Given 利用者がログイン済み When 会議室検索画面で条件を入力して検索する Then 条件に合う会議室の一覧が表示される |
| SPEC-007-02 | | 会議室を予約できる（決済方法としてクレジットカードまたは電子マネーを設定） | | 1. Given 利用者が会議室を選択済み When 予約画面で日時と決済方法を入力して予約する Then 予約が作成され確認画面が表示される |
| SPEC-007-03 | | 予約の変更・取消ができる | | 1. Given 予約済みの利用者 When 予約変更または取消を行う Then 予約が変更または取消される（キャンセルポリシーに準拠） |
| SPEC-007-04 | | 会議室オーナーへ問合せができる | | 1. Given 利用者が会議室詳細画面を閲覧中 When 問合せフォームから質問を送信する Then オーナーに問合せが届く |
| REQ-008 | 利用者が利用した会議室とホストを評価できること | | サービスの品質向上とオーナーの改善のため、利用者からの評価フィードバックの仕組みが必要である | |
| SPEC-008-01 | | 利用した会議室とそのホスト（オーナー）の評価を登録できる | | 1. Given 利用が完了した予約 When 利用者が評価画面で会議室とホストの評価を入力する Then 評価が登録され会議室評価に反映される |

| SPEC-009-01 | | 予約時に決済手段を登録する | | 1. Given 利用者が予約を行う When 決済手段を選択する Then 決済手段が予約に紐づいて保存される |

| SPEC-009-03 | | 月末に会議室オーナーに利用料が支払われる | | 1. Given 月末 When 精算処理が実行される Then オーナーに対して利用料（手数料控除後）が支払われる |

| SPEC-010-01 | | 利用者が会議室に行き、オーナーから鍵を受け取ることで利用開始となる | | 1. Given 予約確定済みの利用者が会議室に到着 When オーナーから鍵を受け取る Then 会議室利用が開始状態になる |
| SPEC-010-02 | | 会議後に鍵を返却したら利用終了となる | | 1. Given 利用中の会議室 When 利用者が鍵を返却する Then 会議室利用が終了状態になる |
