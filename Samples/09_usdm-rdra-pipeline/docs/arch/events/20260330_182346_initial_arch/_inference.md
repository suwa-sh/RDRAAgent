## RDRA/NFR モデル分析結果

### 分析した RDRA 要素

| モデル | 要素数 | 主な特徴 |
|--------|--------|---------|
| BUC | 7業務・10フロー・約30UC | オーナー管理、会議室管理、予約、貸出、評価、運営、精算の7業務。精算にバッチ処理あり |
| アクター | 3種（利用者・オーナー・運営） | 外部2種（利用者・オーナー）+内部1種（運営）。C2Cシェアリングモデル |
| 外部システム | 1種（決済機関） | 精算・利用料の決済処理。冪等性確保が必須 |
| 情報 | 15種 | 個人情報（オーナー・利用者）、金銭情報（予約・手数料・売上・精算）、状態管理（予約・問合せ・オーナー申請） |
| 状態 | 3モデル・18遷移 | 予約状態（7遷移）、オーナー申請状態（6遷移）、問合せ状態（5遷移） |
| 条件 | 4種 | 予約変更条件、利用許諾条件、精算条件、オーナー審査条件 |
