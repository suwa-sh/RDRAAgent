# サービスへ問合せする - 利用者向けフロントエンド仕様

## 変更概要

利用者ポータルにサービス問合せ画面を追加する。問合せカテゴリー選択とテキスト入力のシンプルなフォーム画面。

## 画面仕様

### サービス問合せ画面

- **URL**: `/inquiries/service/new`
- **アクセス権**: 利用者（userロール）
- **ポータル**: user

#### 表示要素とコンポーネントマッピング

| 要素 | 種別 | デザインシステムコンポーネント | 説明 |
|------|------|------------------------------|------|
| ページタイトル | テキスト | Heading（h1） | 「サービスへのお問合せ」 |
| 問合せカテゴリー | セレクト | Select | 使い方/予約・変更・取消/請求・精算/技術的な問題/その他 |
| 問合せ内容 | テキストエリア | Textarea | 問合せ内容（1〜1000文字）、文字数カウンター付き |
| 添付ファイル | ファイルアップロード | FileUpload | スクリーンショット等（任意、5MB以内、jpg/png/pdf） |
| 送信ボタン | ボタン | Button（primary） | 問合せを送信。送信中は無効化 |
| 完了メッセージ | アラート | Alert（success） | 送信完了後に表示 |
| 問合せ履歴リンク | リンク | Link | 送信後に「問合せ履歴を確認する」リンクを表示 |

#### デザイントークン参照

| 用途 | トークン | 値 |
|------|---------|---|
| 背景色 | var(--semantic-background) | #F8FAFC |
| アクセント | var(--portal-primary) | #2563EB |
| 成功アラート背景 | var(--semantic-success-bg) | #F0FDF4 |
| 成功アラートテキスト | var(--semantic-success) | #16A34A |

#### UIロジック

- **状態管理**: フォームの入力値（category, content, attachments）をローカル状態で管理。送信完了後は状態をリセット
- **バリデーション**: 問合せ内容は1文字以上1000文字以内（リアルタイム文字数カウント）。送信時にフロントエンドで検証してからAPI呼び出し
- **ローディング**: 送信ボタン押下後はスピナーを表示しボタンを無効化（二重送信防止）
- **エラーハンドリング**: APIエラー時はフォーム上部にエラーアラートを表示。ネットワークエラーの場合はリトライ可能なメッセージを表示

#### 操作フロー

1. 利用者が `/inquiries/service/new` にアクセスする
2. 問合せカテゴリーをセレクトから選択する（例: 「予約・変更・取消」）
3. テキストエリアに問合せ内容を入力する（文字数リアルタイムカウント）
4. 必要に応じてスクリーンショットなどのファイルを添付する
5. 「送信」ボタンをクリックする（スピナー表示・ボタン無効化）
6. 完了メッセージ「お問合せを受け付けました」とともに問合せ履歴へのリンクが表示される

## コンポーネント設計

### ServiceInquiryForm

- **ベースコンポーネント**: FormCard
- **Props**:
  | Prop | 型 | 必須 | 説明 |
  |------|---|------|------|
  | onSuccess | () => void | No | 送信成功時のコールバック |
- **状態**: category(string), content(string), attachments(File[]), isSubmitting(boolean), isSuccess(boolean), error(string\|null)
- **イベント**: onCategoryChange, onContentChange, onFileAdd, onFileRemove, onSubmit
