# 会議室を登録する - 利用者・オーナー向けフロントエンド仕様

## 変更概要

会議室登録画面を新規作成する。会議室種別（物理/バーチャル）に応じて入力項目を動的に切り替え、会議室情報の入力・送信を行う画面。

## 画面仕様

### 会議室登録画面

- **URL**: `/owner/rooms/new`
- **アクセス権**: ログイン済み会議室オーナー（登録済み状態）
- **ポータル**: owner

#### 表示要素とコンポーネントマッピング

| 要素 | 種別 | デザインシステムコンポーネント | 説明 |
|------|------|------------------------------|------|
| 会議室種別選択 | フォーム | Input（セレクト） | 「物理」「バーチャル」の選択。切り替えで下のフィールドが変化 |
| 会議室名入力 | フォーム | Input (variant: default) | 必須。最大100文字 |
| 所在地入力（物理のみ） | フォーム | Input (variant: default) | 必須（物理）。都道府県〜番地 |
| 広さ入力（物理のみ） | フォーム | Input (variant: default) | 数値（㎡） |
| 収容人数入力（物理のみ） | フォーム | Input (variant: default) | 必須（物理）。数値（人） |
| 価格入力 | フォーム | Input (variant: default) | 必須。数値（円/時間）。PriceDisplay で確認表示 |
| 価格プレビュー | テキスト | PriceDisplay | 入力した価格をリアルタイムでフォーマット表示 |
| 会議ツール種別選択（バーチャルのみ） | フォーム | Input（セレクト） | 「Zoom」「Teams」「Google Meet」 |
| 同時接続数入力（バーチャルのみ） | フォーム | Input (variant: default) | 必須（バーチャル）。数値（人） |
| 録画可否選択（バーチャルのみ） | フォーム | Input（チェックボックス） | 録画可/不可の選択 |
| 設備・機能選択 | フォーム | Input（チェックボックス群） | プロジェクター・ホワイトボード等の複数選択 |
| 登録ボタン | ボタン | Button (variant: default, size: lg) | 必須項目入力後に活性化 |

#### UIロジック

- **状態管理**: 会議室種別（物理/バーチャル）、フォーム値、バリデーション状態、送信中フラグ
- **バリデーション**: 会議室名・価格は必須。物理：所在地・収容人数が追加必須。バーチャル：会議ツール種別・同時接続数が追加必須
- **ローディング**: 登録ボタン押下後はスピナー表示
- **エラーハンドリング**: 400は各フィールドのエラーメッセージを表示

#### 操作フロー

1. `/owner/rooms/new` を開く
2. 会議室種別（物理/バーチャル）を選択する
3. 種別に応じたフォーム項目が表示される
4. 会議室情報を入力する
5. 入力完了後、登録ボタンが活性化される
6. 登録ボタンをクリックする
7. 成功時: 「登録が完了しました。運用ルールを設定して公開準備を進めてください。」と運用ルール設定画面へのリンクを表示

## コンポーネント設計

### RoomRegistrationForm

- **ベースコンポーネント**: Input, Button, PriceDisplay, Card
- **Props**:
  | Prop | 型 | 必須 | 説明 |
  |------|---|------|------|
  | onSuccess | (roomId: string) => void | Yes | 登録成功コールバック |
- **状態**:
  - `roomType`: 'physical' \| 'virtual'
  - `formValues`: RoomFormValues
  - `errors`: Record\<string, string\>
  - `isSubmitting`: boolean
- **イベント**: `onTypeChange`（種別変更）, `onSubmit`（フォーム送信）
