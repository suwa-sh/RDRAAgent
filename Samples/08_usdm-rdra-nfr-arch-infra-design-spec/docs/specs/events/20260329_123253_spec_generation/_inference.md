## 非同期イベント
1. **精算支払連携**: 精算を実行する → 決済機関（外部システム）への支払い連携
2. **会議URL自動通知**: 会議URLを通知する → バーチャル会議室予約確定時にメール等で自動通知
3. **バーチャル会議室利用開始**: タイマートリガー → 利用開始時刻到来で自動遷移
4. **バーチャル会議室利用終了**: タイマートリガー → 利用終了時刻到来で自動遷移

## 全体横断設計方針

### ユーザーフロー
- **利用者フロー**: 会議室検索 → 詳細確認 → 予約申請 → 決済設定 → (許諾待ち) → 利用 → 評価
- **オーナーフロー**: オーナー登録 → 会議室登録 → 運用ルール設定 → 公開 → 予約審査 → 鍵貸出/返却 → 利用者評価 → 精算確認
- **管理者フロー**: オーナー審査 → 利用状況分析 → 手数料売上分析 → 精算計算・実行 → 問合せ対応

### 情報アーキテクチャ
- **利用者ポータル（user）**: /rooms（検索）, /rooms/:id（詳細）, /rooms/:id/book（予約）, /reservations（予約一覧）, /reservations/:id/review（評価）, /inquiries（問合せ）
- **オーナーポータル（owner）**: /owner/rooms（会議室管理）, /owner/reservations（予約管理）, /owner/settlements（精算）, /owner/inquiries（問合せ）, /owner/reviews（評価）
- **管理者ポータル（admin）**: /admin/owners（オーナー管理）, /admin/analytics（分析）, /admin/settlements（精算）, /admin/inquiries（問合せ）

### データ可視化対象
- 手数料売上分析画面: 売上推移チャート（折れ線）、会議室別売上（棒グラフ）
- 利用状況分析画面: 利用回数推移（折れ線）、稼働率（ドーナツ）、会員別利用（テーブル）
- 売上実績確認画面: 売上推移（折れ線）、利用回数（棒グラフ）
- 利用回数確認画面: 利用回数推移（棒グラフ）

## NFR 反映事項
- **可用性**: 24h運用 → 全画面にエラー/ローディング状態定義
