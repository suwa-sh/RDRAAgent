# オーナーへ問合せする - 利用者・オーナー向けフロントエンド仕様

## 変更概要

オーナー問合せ画面を実装する。問合せ内容を入力して送信し、問合せスレッドに表示する。

## 画面仕様

### オーナー問合せ画面

- **URL**: `/rooms/{room_id}/inquiry`
- **アクセス権**: 利用者（ログイン必須）
- **ポータル**: user

#### 表示要素とコンポーネントマッピング

| 要素 | 種別 | デザインシステムコンポーネント | 説明 |
|------|------|------------------------------|------|
| 問合せ入力フィールド | フォーム | Input (type=textarea) | 問合せ内容（1〜500文字） |
| 「送信する」ボタン | ボタン | Button (variant=primary) | 問合せ送信 |
| 問合せ履歴スレッド | リスト | InquiryThread | 過去の問合せ・回答を時系列表示 |
| 問合せ状態バッジ | バッジ | Badge (variant=warning/success) | 未対応=warning、回答済み=success |

#### デザイントークン参照

| 用途 | トークン | 値 |
|------|---------|---|
| 背景色 | var(--semantic-background) | #FFFFFF |
| アクセント | var(--portal-primary) | #2563EB |

#### UIロジック

- **状態管理**: 問合せ内容・送信中フラグ
- **バリデーション**: 問合せ内容は1〜500文字必須
- **ローディング**: 送信ボタン押下後はスピナー・ボタン無効化
- **エラーハンドリング**: API エラー時はトースト通知

#### 操作フロー

1. 会議室詳細画面から「オーナーに問合せる」ボタンをクリック
2. オーナー問合せ画面（/rooms/{room_id}/inquiry）に遷移
3. 問合せ内容を入力
4. 「送信する」ボタンを押す
5. 完了後「問合せを送信しました」メッセージが表示される

## コンポーネント設計

### InquiryForm

- **ベースコンポーネント**: InquiryThread, Input, Button
- **Props**:
  | Prop | 型 | 必須 | 説明 |
  |------|---|------|------|
  | roomId | string | Yes | 会議室ID |
  | ownerId | string | Yes | オーナーID |
  | pastInquiries | Inquiry[] | No | 過去の問合せ履歴 |
  | onSubmit | (content: string) => Promise<void> | Yes | 送信コールバック |
- **状態**: 問合せ内容・送信中フラグ
- **イベント**: onSubmit
