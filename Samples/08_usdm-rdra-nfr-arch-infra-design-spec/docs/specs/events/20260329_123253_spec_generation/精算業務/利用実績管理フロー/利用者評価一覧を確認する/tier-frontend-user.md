# 利用者評価一覧を確認する - 利用者・オーナー向けフロントエンド仕様

## 変更概要

会議室オーナーが自身に対する利用者評価を一覧で確認するための画面を追加する。平均スコアのサマリーと個別評価カード（評価スコア・コメント・評価日時）をページネーション付きで表示する。

## 画面仕様

### 利用者評価一覧画面

- **URL**: `/owner/user-reviews`
- **アクセス権**: 会議室オーナー（ownerロール）
- **ポータル**: owner

#### 表示要素とコンポーネントマッピング

| 要素 | 種別 | デザインシステムコンポーネント | 説明 |
|------|------|------------------------------|------|
| ページタイトル | テキスト | - | 「利用者評価一覧」見出し（2xl） |

| 個別星評価 | データ表示 | StarRating（readOnly, sm） | 個別評価スコアを星で表示 |
| ページネーション | ナビゲーション | Button（outline, sm） | 前へ・次へページ切替ボタン |

#### UIロジック

- **状態管理**: 現在ページ（page: number）をローカルステートで管理。ページ変更時に API を再フェッチする（per_page=20固定）
- **バリデーション**: なし（参照専用画面）
- **ローディング**: API フェッチ中は評価カードエリアにスケルトンスクリーン（3枚分）を表示する
- **エラーハンドリング**: API エラー（500系）時は「データの取得に失敗しました。しばらくしてから再試行してください。」のトースト通知を表示する

#### 操作フロー

1. オーナーポータルサイドメニューから「利用者評価」を選択する
2. 平均スコアバナーに総合平均スコア（星表示）・総評価件数が表示される
3. 個別評価カードが最新評価日時順で20件表示される
4. 20件を超える場合はページネーションで「次へ」ボタンをクリックして次ページを表示する
5. 評価0件の場合は「まだ評価がありません」というメッセージを表示する

## コンポーネント設計

### UserReviewCard

- **ベースコンポーネント**: Card（default）
- **Props**:
  | Prop | 型 | 必須 | 説明 |
  |------|---|------|------|
  | reviewId | string | Yes | 評価ID |
  | userId | string | Yes | 利用者ID |
  | score | number | Yes | 評価スコア（1〜5） |
  | comment | string | No | 評価コメント（なければ「コメントなし」表示） |
  | reviewedAt | string | Yes | 評価日時（ISO 8601形式） |
- **状態**: なし（純粋表示コンポーネント）
- **イベント**: なし

### AverageScoreBanner

- **ベースコンポーネント**: Card（default）
- **Props**:
  | Prop | 型 | 必須 | 説明 |
  |------|---|------|------|
  | averageScore | number | Yes | 平均評価スコア（小数点1位まで表示） |
  | totalCount | number | Yes | 総評価件数 |
- **状態**: なし（純粋表示コンポーネント）
- **イベント**: なし
