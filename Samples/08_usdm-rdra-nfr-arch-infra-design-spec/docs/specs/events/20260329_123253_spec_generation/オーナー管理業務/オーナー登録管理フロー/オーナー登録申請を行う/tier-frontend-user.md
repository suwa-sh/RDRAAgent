# オーナー登録申請を行う - 利用者・オーナー向けフロントエンド仕様

## 変更概要

オーナー登録申請フォーム画面を新規作成する。未認証ユーザー（会議室オーナー候補）が氏名・連絡先・メールアドレス等の申請情報を入力し送信できる画面。

## 画面仕様

### オーナー登録申請画面

- **URL**: `/owner/register`
- **アクセス権**: 未認証ユーザー（新規オーナー登録希望者）
- **ポータル**: owner

#### 表示要素とコンポーネントマッピング

| 要素 | 種別 | デザインシステムコンポーネント | 説明 |
|------|------|------------------------------|------|
| 氏名入力フィールド | フォーム | Input (variant: default) | 必須。姓名スペース区切り。最大50文字 |
| 連絡先入力フィールド | フォーム | Input (variant: default) | 必須。電話番号形式 |
| メールアドレス入力フィールド | フォーム | Input (variant: default) | 必須。メール形式バリデーション |
| 申請ボタン | ボタン | Button (variant: default, size: lg) | 全項目入力後に活性化 |
| エラーメッセージ表示 | テキスト | Badge (variant: destructive) | バリデーションエラー時に各フィールド下に表示 |
| 申請完了メッセージ | テキスト | Card (variant: default) | 申請成功後に表示 |

#### デザイントークン参照

| 用途 | トークン | 値 |
|------|---------|---|
| 背景色 | var(--semantic-background) | #FFFFFF |
| アクセント | var(--portal-primary) | #0D9488 (Teal 600) |
| エラー色 | var(--semantic-destructive) | #DC2626 |
| ボーダー色 | var(--semantic-border) | #E2E8F0 |

#### UIロジック

- **状態管理**: フォームの各フィールドの値、バリデーション状態、送信中フラグ、送信完了フラグを管理
- **バリデーション**: 全フィールド必須。メールアドレスはRFC 5322準拠の形式チェック。電話番号は数字・ハイフン・括弧のみ許可
- **ローディング**: 申請ボタン押下後はスピナーを表示し重複送信を防止
- **エラーハンドリング**: 409（メール重複）は「このメールアドレスは既に登録されています」を表示。500系は「しばらくしてから再度お試しください」を表示

#### 操作フロー

1. オーナー登録申請画面（`/owner/register`）を開く
2. 氏名・連絡先・メールアドレスを入力する
3. 入力値のフロントエンドバリデーションが実行される
4. バリデーション通過後、申請ボタンが活性化される
5. 申請ボタンをクリックする
6. ローディングスピナーが表示される
7. API レスポンスを受け取る
8. 成功時: 申請完了メッセージ（Card）を表示する
9. エラー時: エラーバッジを該当フィールド下に表示する

## コンポーネント設計

### OwnerRegistrationForm

- **ベースコンポーネント**: Input, Button, Card
- **Props**:
  | Prop | 型 | 必須 | 説明 |
  |------|---|------|------|
  | onSuccess | () => void | Yes | 申請成功時のコールバック |
- **状態**:
  - `formValues`: { name: string, phone: string, email: string }
  - `errors`: { name?: string, phone?: string, email?: string }
  - `isSubmitting`: boolean
  - `isSubmitted`: boolean
- **イベント**: `onSubmit`（フォーム送信）, `onChange`（フィールド変更）
