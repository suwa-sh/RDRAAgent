# 利用者を評価する - 利用者・オーナー向けフロントエンド仕様

## 変更概要

利用者評価登録画面を実装する。StarRatingコンポーネントで5段階評価を入力し、コメントテキストエリアと合わせて送信する。利用終了後の評価登録フローをサポートする。

## 画面仕様

### 利用者評価登録画面

- **URL**: `/owner/usages/:id/user-rating`
- **アクセス権**: 会議室オーナー（該当会議室の所有者のみ）
- **ポータル**: owner（オーナーポータル）

#### 表示要素とコンポーネントマッピング

| 要素 | 種別 | デザインシステムコンポーネント | 説明 |
|------|------|------------------------------|------|
| 利用者名 | テキスト | - (セマンティックHTML h2) | 評価対象の利用者名 |
| 利用情報サマリー | カード | Card (variant: default) | 利用日時・利用時間・会議室名 |
| 評価スコア入力 | 評価入力 | StarRating (readOnly: false, size: lg) | インタラクティブな5段階評価入力 |
| コメント入力 | フォーム | Input (variant: default, multiline) | 評価コメント（最大500文字、任意） |
| 送信ボタン | ボタン | Button (variant: primary) | 「評価を送信」ボタン |
| スキップボタン | ボタン | Button (variant: ghost) | 「今回は評価しない」（未評価のまま完了） |

#### デザイントークン参照

| 用途 | トークン | 値 |
|------|---------|---|
| 背景色 | var(--semantic-background) | #FFFFFF |
| アクセント | var(--portal-primary) | #0D9488 (Teal 600, オーナーポータル) |
| 評価星色 | var(--semantic-rating) | #FBBF24 |

#### UIロジック

- **状態管理**: score（number | null）、comment（string）、isSubmitting（boolean）をローカルステートで管理
- **バリデーション**: スコア未選択の場合「評価スコアを選択してください」バリデーションエラーを表示。コメントは任意（最大500文字）
- **ローディング**: 送信中は「送信ボタン」をdisabled化しスピナーを表示
- **エラーハンドリング**: 409エラーは「評価済みです」モーダルを表示。サーバーエラーはトーストで通知

#### 操作フロー

1. 鍵返却記録完了後の画面から「利用者を評価する」リンクをクリック、または利用済み予約一覧から遷移
2. 利用情報サマリー（田中太郎、2026-03-29 10:00-12:00、渋谷会議室001）を確認
3. StarRatingコンポーネントでスコアを選択（1-5）
4. コメントを任意入力
5. 「評価を送信」ボタンをクリック
6. 送信完了後、「評価を送信しました」トーストが表示され予約一覧に遷移

## コンポーネント設計

### UserRatingForm

- **ベースコンポーネント**: StarRating + Input + Card
- **Props**:
  | Prop | 型 | 必須 | 説明 |
  |------|---|------|------|
  | usageId | string | Yes | 会議室利用ID |
  | usage | UsageSummary | Yes | 利用サマリー（利用者名・日時） |
  | onSuccess | () => void | Yes | 評価送信成功後のコールバック |
  | onSkip | () => void | Yes | 評価スキップのコールバック |
- **状態**: idle, submitting, done
- **イベント**: onSubmit（score + comment）、onSkip
