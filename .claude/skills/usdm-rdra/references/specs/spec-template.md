# Spec テンプレート定義

UC 単位の仕様ドキュメントのフォーマットを定義する。

## ディレクトリ構成

```
{UC名}/
  spec.md              # UC の仕様概要
  tier-frontend.md     # フロントエンド仕様
  tier-backend.md      # バックエンド仕様
  tier-infra.md        # インフラ仕様（必要な場合のみ）
```

## spec.md フォーマット

```markdown
# {UC名}

## 概要

{UC の目的と概要を1-3文で記述}

## 関連 RDRA モデル

| モデル種別 | 要素名 | 関連 |
|-----------|--------|------|
| BUC | {BUC名} | このUCを含むBUC |
| アクター | {アクター名} | 操作するアクター |
| 情報 | {情報名} | 参照・更新する情報 |
| 状態 | {状態名} | 関連する状態遷移 |
| 条件 | {条件名} | 適用される条件 |
| 外部システム | {外部システム名} | 連携する外部システム |

## E2E 完了条件（BDD）

### 正常系

```gherkin
Feature: {UC名}

  Scenario: {シナリオ名}
    Given {前提条件}
    When {操作}
    Then {期待結果}

  Scenario: {シナリオ名2}
    Given {前提条件}
    When {操作}
    Then {期待結果}
```

### 異常系

```gherkin
  Scenario: {異常シナリオ名}
    Given {前提条件}
    When {異常操作}
    Then {エラーハンドリング結果}
```

## ティア別仕様

- [フロントエンド](tier-frontend.md)
- [バックエンド](tier-backend.md)
- [インフラ](tier-infra.md)（該当する場合）
```

## tier-frontend.md フォーマット

```markdown
# {UC名} - フロントエンド仕様

## 変更概要

{フロントエンドで必要な変更の概要}

## 画面仕様

### {画面名}

- **URL**: {画面のパス}
- **アクセス権**: {アクセスできるアクター}

#### 表示要素

| 要素 | 種別 | 説明 |
|------|------|------|
| {要素名} | テキスト/ボタン/テーブル/フォーム | {説明} |

#### 操作フロー

1. {操作ステップ1}
2. {操作ステップ2}
3. ...

## ティア完了条件（BDD）

```gherkin
Feature: {UC名} - フロントエンド

  Scenario: {画面表示シナリオ}
    Given {前提条件}
    When {画面操作}
    Then {画面の期待状態}
```
```

```markdown
# {UC名} - バックエンド仕様

## 変更概要

{バックエンドで必要な変更の概要}

- **パス**: {APIパス}
- **認証**: {認証方式}

#### リクエスト

| パラメータ | 型 | 必須 | 説明 |

#### レスポンス

| フィールド | 型 | 説明 |
|-----------|---|------|
| {フィールド名} | string/number/boolean | {説明} |

## データモデル変更

### {テーブル/エンティティ名}

| カラム | 型 | 説明 | 変更種別 |
|--------|---|------|---------|

## ビジネスルール

- {ビジネスルール1}
- {ビジネスルール2}

Feature: {UC名} - バックエンド

```

## tier-infra.md フォーマット

```markdown
# {UC名} - インフラ仕様

## 変更概要

{インフラで必要な変更の概要}

## リソース変更

| リソース | 種別 | 変更内容 |
|---------|------|---------|
| {リソース名} | DB/キャッシュ/キュー/ストレージ | {変更内容} |

## 設定変更

| 設定項目 | 変更前 | 変更後 | 理由 |
|---------|--------|--------|------|
| {設定名} | {変更前の値} | {変更後の値} | {理由} |

## ティア完了条件

- {インフラ完了条件1}
- {インフラ完了条件2}
```

## 注意事項

- tier-infra.md はインフラ変更が必要な場合のみ作成する
- BDD シナリオは具体的な値を含める（「適切な値」のような曖昧表現は避ける）
- 関連 RDRA モデルは `docs/rdra/latest/*.tsv` の実際の要素名を使用する
- spec.md の E2E 完了条件はティア横断のシナリオ、tier-*.md の完了条件はティア内で閉じたシナリオにする
