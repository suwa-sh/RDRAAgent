# ロギングアーキテクチャ リファレンス

本ドキュメントは arch-inference-rules.md から参照され、ロギング方針の推論を支援する。

## ログカテゴリ

| カテゴリ | 説明 | 主な利用者 |
|---------|------|----------|
| アクセスログ | システムへの入口と出口の記録 | 運用担当者、SRE |
| 監査ログ | ビジネス上重要なイベントの不変な記録（誰が、いつ、何を、どうしたか） | セキュリティ担当、監査人 |
| 診断ログ | 内部フロー追跡、デバッグ用 | 開発者 |
| 依存関係ログ | 外部システムとの通信記録 | 開発者、SRE |

## レイヤー別ログ出力方針

### 基本原則

- 各レイヤーは自身の責務に対応するログカテゴリのみを出力する
- **domain 層は直接的なロギングを行わない**（ドメインイベントの発行または例外のスローで状態変化を通知する）
- ログの出力ポイントはレイヤーの境界（入口/出口）に集約する

### レイヤー × ログカテゴリ マトリクス

| レイヤー | アクセスログ | 監査ログ | 診断ログ | 依存関係ログ |
|---------|:----------:|:-------:|:-------:|:----------:|
| presentation | ◎ 主責務 | - | ○ 起動/終了 | - |
| usecase | - | ◎ 主責務 | ○ 機能開始/終了 | - |
| domain | - | - | - | - |
| gateway | - | △ 照合用 | ○ エラー/進捗 | ◎ 主責務 |

◎ = 主責務、○ = 副次的に出力、△ = 特定条件で出力、- = 出力しない

## アーキタイプ別ロギング戦略

### Web アプリケーション（Backend API ティア）

| カテゴリ | レベル | レイヤー | 内容 |
|---------|-------|---------|------|
| アクセスログ | ERROR | presentation | サーバーエラー（5xx）の HTTP メタデータ |
| アクセスログ | WARN | presentation | クライアントエラー（4xx）の HTTP メタデータ |
| アクセスログ | INFO | presentation | 成功した HTTP リクエスト/レスポンスのメタデータ（2xx, 3xx） |
| 監査ログ | INFO | usecase | 重要なビジネスイベントの記録（誰が、何を、どうしたか） |
| 診断ログ | INFO | usecase | 機能の開始・終了 |
| 依存関係ログ | INFO | gateway | 外部 DB/API 呼び出しの開始・終了、処理時間、成否 |
| 依存関係ログ | DEBUG | gateway | SQL クエリ、リクエスト/レスポンス本文 |
| (出力しない) | - | domain | ドメインイベントの発行または例外のスローで通知 |

### バッチジョブ（Worker ティア - CronJob）

| カテゴリ | レベル | レイヤー | 内容 |
|---------|-------|---------|------|
| 診断ログ | INFO | presentation | ジョブ起動、実行パラメータ、ジョブ終了（COMPLETED/FAILED） |
| 監査ログ | INFO | usecase | 重要なビジネスイベントの記録 |
| 診断ログ | INFO | usecase | 総実行時間、処理件数サマリー |
| 依存関係ログ | INFO | gateway | 外部 DB/API 呼び出しの処理時間、成否 |
| 診断ログ | ERROR | gateway | アイテム単位の処理エラー（対象アイテム、例外詳細） |
| 診断ログ | DEBUG | gateway | 定期的な処理進捗（ハートビート） |
| 監査ログ | INFO | gateway | 処理後の照合用集計情報（合計金額、チェックサム等） |
| (出力しない) | - | domain | ドメインイベントの発行または例外のスローで通知 |

### メッセージコンシューマ（Worker ティア - MQ）

| カテゴリ | レベル | レイヤー | 内容 |
|---------|-------|---------|------|
| 診断ログ | INFO | presentation | ストリーム処理の起動・停止 |
| 監査ログ | WARN | usecase | 重要なセキュリティイベントの検出 |
| 診断ログ | ERROR | usecase | 処理不能メッセージ（ポイズンピル）と例外情報 |
| 依存関係ログ | INFO | gateway | 集約されたメッセージ処理メトリクス |
| 依存関係ログ | WARN | gateway | パフォーマンス低下の兆候 |
| (出力しない) | - | domain | ドメインイベントの発行または例外のスローで通知 |

## 構造化ログの必須フィールド

| フィールド | 説明 |
|-----------|------|
| timestamp | ISO 8601 形式 |
| level | INFO, WARN, ERROR, DEBUG, TRACE |
| correlation_id / trace_id | リクエスト横断の追跡 ID |
| service | サービス名 + バージョン |
| message | 静的テキスト（動的値は context に分離） |
| context | 追加情報のキーバリューペア（user_id, order_id 等） |

## セキュリティ制約

ログに記録してはいけない情報:
- 認証情報（パスワード、API キー、アクセストークン）
- PII（マイナンバー、健康情報、メールアドレス、電話番号）
- 金融データ（クレジットカード番号、銀行口座情報）

必要時はマスキング/ハッシュ化を適用する。

## RDRA/NFR シグナルとの対応

| シグナル | ロギング方針への影響 |
|---------|------------------|
| NFR C.6.1 ログ保管期間 | 保管期間の設定。監査ログは長期保管 |
| NFR C.1.3 監視範囲 | アクセスログ + 依存関係ログの出力レベル |
| NFR E.7.1 監査ログ | 監査ログの出力範囲と保管ポリシー |
| NFR E.6.1 データ暗号化 | PII のマスキング/ハッシュ化ルール |
| 外部システム.tsv | 依存関係ログの出力対象 |
| 状態.tsv の状態遷移 | 監査ログの出力対象イベント |
