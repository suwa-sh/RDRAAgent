# RDRASpec

## 実施内容

### 出力条件
- 各プロンプトの出力はプロジェクト直下の「2_RDRASpec」フォルダーの配下に出力する
- 「RDRA_Knowledge」フォルダー配下には絶対に出力しない

## 並列実行の前提条件
- 各phase内のプロンプトは**完全に独立**しており、相互依存関係がない
- 各プロンプトは異なる出力ファイルを生成するため、ファイル衝突は発生しない
- すべてのプロンプトは同じ入力ファイル（`1_RDRA/関連データ.txt`など）を**読み取り専用**で参照する

## phase1の実行
### 実行プロンプトと出力ファイル:
1. `RDRA_Knowledge/2_RDRASpec/phase1/21_論理データ生成.md`
   → 出力: `2_RDRASpec/論理データモデル.md`
2. `RDRA_Knowledge/2_RDRASpec/phase1/22_ビジネスルール生成.md`
   → 出力: `2_RDRASpec/ビジネスルール.md`
3. `RDRA_Knowledge/2_RDRASpec/phase1/23_画面一覧生成.md`
   → 出力: `2_RDRASpec/phase1/画面一覧.json`
4. `RDRA_Knowledge/2_RDRASpec/phase1/24_BUC画面生成.md`
   → 出力: `2_RDRASpec/phase1/BUC画面.json`
5. `RDRA_Knowledge/2_RDRASpec/phase1/25_アクター画面生成.md`
   → 出力: `2_RDRASpec/phase1/アクター画面.json`

### 実行の前提:
- 5個のプロンプトファイルを同じツール呼び出しバッチで読み込む
- 各プロンプトの指示に従って、5個のファイルを並行して生成する
- 出力ファイル名が異なるため、並列実行による競合は発生しない
- 各プロンプトは独立して実行可能（依存関係なし）
- すべてのプロンプトは`1_RDRA/関連データ.txt`を読み取り専用で参照する

## phase2の実行
前提条件: phase1のすべてのファイルが生成完了していること

### 実行プロンプトと出力ファイル:
